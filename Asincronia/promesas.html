Asincronia

La asincronía en JavaScript es esencial para construir aplicaciones ágiles sin bloquear la ejecución. Aunque el lenguaje corre en un solo hilo (single thread), el event loop permite programar tareas como llamadas a API, timers y lecturas de archivos sin detener el flujo principal. Aquí verás cómo se comporta el código y cómo usar callbacks y promesas con then/catch para un manejo claro de resultados y errores.

La ejecución principal avanza línea a línea, pero ciertas operaciones se delegan y vuelven “más tarde”. Con setTimeout, por ejemplo, programas una tarea futura y el programa no espera: sigue. Por eso verás primero el flujo sincrono, y finalmente el resultado del timeout cuando se cumpla el tiempo. Nada se bloquea; la tarea se agenda y se resuelve después en la pila de ejecución.

Promesas

Las promesas son objetos que representan el eventual resultado (completado o fallido) de una operación asincrónica. Funciona al indicar al navegador que ciertas funciones deben ejecutarse de manera independiente (como peticiones HTTP, temporizadores, lectura de archivos, etc) permitiendo trabajar de forma asincrona. Una vez resueltas, las promesas se integran en el call stack para proporcionar un resultado.

Estados de una promesa
  • Pending (pendiente): Este es el estado inicial. La promesa está creada pero aún no se ha resuelto ni ha fallado. Cuando deja de estar pendiente, almacena un valor (si fue resuelta) o una razón (si fue rechazada).
  • Fulfilled (resuelta): Indica que la promesa se ha completado con éxito. Se gestiona a travez del then.
  • Rejected (rechazada): Ocurre cuando la promesa no se puede cumplir, produciendo un error. Se gestiona a travez del catch.

El Call Stack (pila de llamadas) es una pila de tareas donde se registran las funciones que deben ejecutarse. Cuando se llama a una funcion esta se agrega al callstack, ejecutandose de manera secuencial, a esto se le conoce como ejecucion sincrona.

Una Operacion asincrona es una tarea que se inicia y luego se ejecuta en un momento posterior, sin bloquear el hilo principal de ejecución. Esto permite que el programa continúe ejecutando otro código mientras la operación asincrónica se completa.

Promise

Una promesa se crea utilizando el constructor Promise(). Dentro del constructor, se pasa una función asíncrona como argumento. Esta función asíncrona, también conocida como función de resolución o rechazo, recibe dos argumentos:

    Funciones callback que usan las promesas
  • resolve una función de Javascript que se llama cuando la operación asincrónica se completa con éxito. Se le pasa un valor como parámetro, que representa el resultado de la operación.
  • reject una función de Javascript que se llama cuando la operación asincrónica falla. Se le pasa un error como parámetro, que representa la causa del fallo.

Sintaxis

new Promise(function(resolve, reject) {
  codigo asincrono
})

  • new Promise crea una nueva promesa usando el constructor (Promise) el cual devuelve un objeto que representa la eventual finalizacion o falla de una operacion asincrona.
  • function(resolve, reject) esta funcion es el argumento del constructor "Promise", recibe dos parametros que representan dos funciones de Javascript. Estos callbacks seran llamados en medio de la funcion asincrona y reciben un valor que luego retornan, este valor representa el resultado de la operacion segun sea el caso: exito o error, y se mostrara en el estado de la promesa.
  • codigo asincrono aqui va la logica asincrona, y una condicion que pregunta si la condicion se cumple llamamos a resolve y si no llamamos a reject.
  • Promise devuelve una instancia del objeto "Promise", este objeto no guarda un valor, sino mantiene el estado y comportamiento que permiten gestionar operaciones asincronas. Solo es visible cuando se manipula con metodos como: "then", "catch", "finally".

Metodos que usan las promesas

then (entonces) Es un metodo de las promesas que recibe uno o dos callback como argumento que se ejecutan si la promesa se resuelve. Estos callbacks los llama javascript con el valor correspondiente una vez que la operación asíncrona (la promesa) termina. then devuelve una nueva promesa que permite encadenar operaciones asincrónicas. A través de este método, podemos llevar a cabo acciones basadas en el resultado exitoso de una promesa.

  • function(resultado) es un callback conocido como funcion de exito que se ejecuta si la promesa devuelve un resultado exitoso. then puede recibir dos callbacks: uno para exito, y otro para error, sin embargo es mas comun usar catch() para el manejo de errores.
  • function(error) este segundo callback tambien conocido como funcion de rechazo, puede atrapar errores si estos ocurren en el bloque de codigo que esta dentro del mismo then(). recibe el error en su parametro, es el callback que la promesa llamará si se rechaza.
  • resultado es un parámetro (una variable local) que representa el valor con el que se resolvió la promesa (el valor de éxito o error).

catch (atrapar) Se utiliza para manejar cualquier error en la cadena. Si la promesa falla, el catch nos proporciona la información necesaria para entender por qué no se resolvió.

async y await

asincrono/esperar es un paradigma que nos permite escribir código asíncrono de forma más legible y estructurada. En JavaScript, el uso de async/await nos permite manejar promesas de una manera más clara, comparada con los métodos tradicionales que usan .then() y .catch(). Cuando declaras una función con async, esta automáticamente devuelve una promesa. Por otro lado, await hace que JavaScript espere a que una promesa se resuelva antes de continuar con el siguiente bloque de código.

  • async es una palabra clave que se coloca antes de una funcion, hace que esa funcion siempre devuelva una promesa, si la funcion devuelve un valor normal, javascript lo envuelve automaticamente en una promesa resuelta.
  • await solo puede usarse dentro de una funcion async, su funcion es pausar la ejecucion de la funcion hasta que la promesa se resuelva o rechace, haciendo que el codigo asincrono se lea como si fuera sincrono.

Sintaxis

async function nombreFuncion() {
  try {
   const resultado = await algunaPromesa()
   codigo para manejar el resultado
  } catch (error) {
   codigo para manejar el error
  }
}

async function fetchData() {

  try {
   const resultado = await fetch('https://swapi.dev/api/planets/3/')
   const datos = await resultado.json()
   console.log(datos)
  } catch (error) {
   console.log(error)
  }

}

Para poner en práctica estos conceptos, vamos a hacer una petición a una API pública usando fetch. Este método proporciona una forma fácil de obtener recursos de manera asíncrona a través de la red.

  • async convierte la funcion "fetchData" en asincrona y devuelve una promesa.
  • try maneja el codigo del fetch, se ejecuta miestras el codigo de peticion se resuelva, y si no catch lo atrapa y se ejecuta su codigo.
  • await detiene el codigo hasta que se resuelva el fetch, el segundo "await" es necesario para que "datos" se vaya tambien al Queue y se ejecute de manera posterior a "respuesta" cuando obtenga los datos.
  • fetch realiza una solicitud a la API de Star Wars, una vez recibida la respuesta, la convierte a JSON y luego imprime los datos.

Manejo de Errores con try / catch

(intentar/atrapar) es una estructura de control que permiten gestionar los errores en codigo sincrono y asincrono de manera controlada. Esto permite que el programa continúe ejecutándose incluso después de un error, evitando que todo el script falle.

El bloque try es donde se coloca el codigo que pueda fallar durante su ejecucion, puede contener uno o mas bloques try y termina con al menos una clausala catch o finally.

El bloque catch se ejecuta si el codigo en try lanza un error, capturandolo en una variable que recibe como parametro. Cuando no se utiliza el error no se detecta y continuara en el bloque superior try.

try{
  let result = 3 + z
} catch (error) {
  consola.innerHTML =
  `ha ocurrido el siguiente error: ${error.message}`
}

Metodos de Peticion

Cuando hablamos de métodos de petición (o HTTP request methods), nos referimos a las acciones que un cliente (ej. tu navegador o una app) le pide a un servidor que realice sobre un recurso. En respuesta, el servidor provee la información o realiza acciones solicitadas, lo que permite al cliente renderizar páginas, actualizar datos o recibir contenido nuevo. Los metodos de peticion son conocidos como los “verbos” del lenguaje HTTP, y cada uno tiene un propósito distinto.

HTTP significa HyperText Transfer Protocol (Protocolo de Transferencia de Hipertexto). Es el lenguaje estándar que usan los navegadores (clientes) y los servidores para comunicarse e intercambiar información en la web. HTTPS es una versión más segura o una extensión de HTTP. En HTTPS, el navegador y el servidor establecen una conexión segura y cifrada antes de transferir datos.

  • Permite que tu navegador pida páginas, imágenes, videos, etc. a un servidor.
  • Define cómo se hacen las peticiones (con métodos como GET, POST, PUT, DELETE).
  • Define cómo se envían las respuestas (con códigos de estado como 200 OK, 404 Not Found, 500 Error).
Funcion de las URL en las solicitudes HTTP

Cada vez que ingresamos una URL en un navegador, como "platzi.com", se inicia un proceso en el que se envía una solicitud HTTP al servidor correspondiente. Este mecanismo es el que permite al cliente obtener la información necesaria para mostrar al usuario la página solicitada. Si todo va bien, el servidor proporciona el contenido; si hay un problema, como un recurso que no se encuentra, devuelve un error 404.

Metodos/Verbos HTTP para peticiones

Los verbos HTTP son esenciales para definir la acción que se desea realizar durante una interacción cliente-servidor. A continuación, exploramos los verbos más comunes:

  • GET Se utiliza para obtener información que ya existe en el servidor sin realizar ningún cambio. Por ejemplo, al acceder a "platzi.com", se envía un GET para recibir y mostrar la página en el navegador.
  • POST Este verbo es empleado para enviar datos nuevos al servidor. Cuando publicamos una actualización en redes sociales, como en Facebook, se usa un POST para almacenar esa nueva información en la base de datos del usuario.
  • PUT Se utiliza para actualizar un recurso existente, reemplazando su contenido por completo.
  • PATCH Se utiliza para actualizar un recurso existente, modificando solo los datos especificados.
  • DELETE Se emplea cuando deseamos eliminar un recurso del servidor. Puede ser la foto que subimos hace años y que ya no queremos conservar.

fetch

Es una función global de JavaScript que permite realizar peticiones HTTP desde el navegador de forma asíncrona y basada en Promesas; esta información la podemos manejar y mostrar en el DOM. Mediante fetch podemos hacer pedidos a una API, que a menudo nos devuelve datos en formato JSON. La Promesa representa un valor que podría estar disponible ahora, en el futuro, o nunca. Mientras la solicitud viaja por la red (tarda un tiempo), el resto de tu código JavaScript continúa ejecutándose.

Sintaxis

fetch(URL, opciones{})
   .then(response => codigo para manejar respuesta)
   .catch(error => codigo para manejar error)

  • fetch() realiza una peticion http y devolverá una promise que será aceptada cuando reciba una respuesta y sólo será rechazada si hay un fallo de red o si por alguna razón no se pudo completar la petición.
    • Argumentos que recibe por parametros:
  • URL en este argumento especificamos una cadena con la direccion del recurso del que queremos obtener datos.
  • opciones{} este parametro es opcional, recibe un objeto en donde podemos especificar algunas caracteristicas del fetch como:
    • method aqui se define el metodo HTTP de la peticion (como string y en mayusculas). Por defecto usa "GET", pero tambien podemos especificar los otros metodos: "POST", "PUT" etc.
    • headers cabeceras HTTP, metadata como tipo de contenido o autenticación. Por defecto, se define dentro de un objeto {}.
    • body cuerpo de la petición HTTP. Datos enviados; usa JSON.stringify para cumplir el formato esperado. Puede ser de varios tipos: String, FormData, Blob, etc...
    • credentials modo de credenciales. Por defecto, omit. Otras opciones: same-origin e include.

Metodos de fetch

then() es un metodo que sirve para manejar las promesas. Recibe una función callback donde su parámetro response es el objeto de respuesta de la petición que hemos realizado, eso quiere decir que recibe la respuesta que nos llega del servidor web al momento de recibir nuestra petición. En su interior realizaremos la lógica que queramos hacer con la respuesta a nuestra petición.

response (variable) representa un objeto response nativo de Javascript que contiene los metadatos de la respuesta HTTP, Cuando usas fetch(url) en JavaScript, la promesa se resuelve con este objeto Response. Generalmente, el siguiente paso es llamar a un método como .json() sobre este objeto para extraer los datos que están dentro del body. Podemos acceder a los valores del objeto response como lo hariamos con otros objetos.

Propiedad Valor Significado
status 200 El código de estado HTTP. 200 significa "OK" (éxito). La solicitud fue recibida, entendida y aceptada.
ok true Un booleano que es `true` si el código de estado HTTP está en el rango 200-299 (lo que indica éxito). Es una manera rápida de verificar el éxito.
url 'https://jsonplaceholder.typicode.com/posts/1' La URL final de donde se obtuvo la respuesta.
type 'cors' Indica el tipo de respuesta. cors (Cross-Origin Resource Sharing) significa que el recurso se solicitó desde un dominio diferente al de tu aplicación (lo cual es normal para APIs externas).
redirected false Indica si la solicitud fue redirigida a una URL diferente antes de obtener la respuesta final.
headers Headers {} Un objeto que contiene todos los encabezados HTTP que el servidor envió con la respuesta (por ejemplo, `Content-Type`, `Date`, etc.).
body ReadableStream El contenido real (los datos JSON, el HTML, etc.) enviado por el servidor. Para acceder a los datos, necesitas llamar a un método como `response.json()` o `response.text()`, lo que consume este *stream*.
bodyUsed false Indica si el cuerpo (`body`) de la respuesta ya ha sido leído (consumido) por un método como `response.json()`.
El response incluye un status que describe el resultado del request.
  • 200: éxito.
  • 201: creado (se agregó correctamente un recurso).
  • 3xx: redirecciones.
  • 400: error del cliente (permisos o datos faltantes).
  • 401: no autenticado.
  • 404: recurso no encontrado.
  • 500: error interno del servidor.
Puedes inspeccionar el response en el navegador y validar si el producto de prueba se agregó haciendo un GET al endpoint y revisando los últimos elementos.

catch (atrapar) Se utiliza para manejar cualquier error en la cadena. Si la promesa falla, el catch nos proporciona la información necesaria para entender por qué no se resolvió.

fetch( "https://jsonplaceholder.typicode.com/posts/1" )
   .then(response => response.json())
   .then(datos => console.log(datos))

Este código es una cadena de promesas que realiza una solicitud GET a una API de ejemplo que devuelve información ficticia.

  • fetch() recibe como argumento una URL y devuelve una Promise que se resuelve con un objeto response.
  • .then(response) se encadena a la promesa que devuelve fetch y response es el objeto que representa la respuesta HTTP. Se ejecuta solamente cuando la solicitud de red ha terminado y la promesa se ha resuelto (es decir, el servidor a respondido)
  • json() es un metodo clave de objeto response, este lee el cuerpo de la respuesta de forma asincrona (lee response.body ), convierte el cuerpo de la respuesta en un objeto javascript, y retorna una nueva Promise, que se resuelve con otro objeto final (data).

    • ¿Porque devuelve una promesa?

    El método response.json() devuelve una Promesa porque el proceso de leer el cuerpo (body) de la respuesta HTTP y convertirlo de una cadena de texto a un objeto JavaScript utilizable (JSON) es una operación asíncrona que toma tiempo. ⏳

    • 1. El Cuerpo es un "Stream" (Flujo de Datos)

      Cuando recibes una respuesta de red, el cuerpo de la respuesta (response.body) no es un simple objeto ya listo, sino un ReadableStream. Esto significa que los datos pueden llegar en partes a lo largo del tiempo, especialmente si el archivo es grande.
    • 2. Lectura y Conversión

      El método response.json() tiene que realizar dos tareas importantes: Leer todo el stream de datos hasta que el cuerpo esté completo. Analizar (parsear) la cadena de texto completa para convertirla en el objeto JavaScript (JSON) que necesitas.
    • 3. Evitar el Bloqueo del Hilo Principal

      Si estas operaciones (leer el stream y parsear) se ejecutaran de forma síncrona, el hilo principal de tu navegador (donde se ejecuta el JavaScript que maneja la interfaz de usuario) se bloquearía completamente. Esto haría que la página se congelara y no respondiera hasta que se completara el proceso.

      Al devolver una Promesa, response.json() permite que la lectura y el parseo ocurran en segundo plano sin congelar la interfaz. La Promesa simplemente indica: "Te notificaré (a través del siguiente .then()) cuando el trabajo esté hecho y los datos estén listos." De esta forma, el flujo de fetch es completamente asíncrono:

      fetch(url) → Devuelve una Promesa (por la solicitud de red).
      response.json() → Devuelve una Promesa (por la lectura y conversión de datos).
  • .then(datos) este segundo then se encadena para esperar a que la Promesa devuelta por response.json() se resuelva: espera a que el cuerpo de la respuesta haya sido leido y convertido a un objeto Javascript(el paso anterior), datos es el objeto javascript que representa los datos JSON extraidos y lo muestra por consola.
