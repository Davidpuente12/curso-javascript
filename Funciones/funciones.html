Funciones

Es un procedimiento o un conjunto de instrucciones que realiza una tarea o calcula un valor para ello debe tomar alguna entrada y devolver una salida donde haya alguna relacion, para usarla debemos llamarla desde algun lugar de nuestro codigo. Las funciones son de tipo "function" mas sin embargo tambien son "objetos" ya que tienen propiedades, metodos, contexto de ejecucion y pueden ser invocadas.

function

una función es un bloque de código reutilizable que ejecuta una tarea específica o calcula un valor, y que puedes invocar (llamar) en cualquier momento de tu programa. Las funciones son los principales “bloques de construcción” del programa. Permiten que el código se llame muchas veces sin repetición.

Las funciones tambien son tipo de dato especial, conocido como Objeto de primera clase, esto significa que se pueden tratar como un valor, al que podemos asignar a variables, pasar como argumento a otras funciones, y retornar desde funciones. De esta forma las funciones pueden viajan por todo el codigo como un dato que contiene valores utiles para otras partes de nuestro codigo.

Sintaxis

function nombre(parametros) {
  // bloque de instrucciones
  return valor retornado
}

nombre(argumentos) //llamado de la funcion

  • function es la palabra clave que se usa para una declaracion de funcion o una expresion de funcion.
  • Nombre las funciones deben tener un nombre descriptivo acerca de lo que realiza esa funcion, debe estar en "camelCase" (la primera letra minúscula y las iniciales de palabras subsiguientes en mayúsculas). El nombre de la funcion es al mismo tiempo la variable que almacena lo que retorna dicha funcion.
  • Parametros van seguido del nombre de la funcion, entre parentesis y separados por comas, es la variable que guarda un valor (entrada de datos) que podra usar la funcion. Se pueden especificar mas de uno.
  • instruccion{...} es donde se ecribe el codigo que la funcion ejecutara, se encierra entre llaves. Puede incluir un return para devolver un valor al finalizar la ejecucion.
  • return la setencia return finaliza la ejecucion de la funcion y devuelven un valor a donde se llama a la funcion. su uso es opcional pero si no hay return explícito y la funcion no realiza ninguna accion, la función devuelve undefined.
  • Llamado de funcion Para ejecutar una función, se utiliza su nombre seguido de paréntesis que, opcionalmente, contienen argumentos; estos son los valores reales que reemplazan a los parámetros de la funcion.
Parametros por Defecto

consiste en establecer parametros pre-definidos en una funcion, de esta manera podemos garantizar que la funcion cumpla su tarea correctamente en caso de que no se le pasen los argumentos que necesita para trabajar. La forma tradicional era por medio de variables que obtienen el parametro (si se establece en el llamado) de lo contrario toma el valor por defecto gracias al operador or (||).

desde EcmaScript 6, se eliminan las declaraciones de variables y el operador or, y asignamos el valor por defecto en los mismos parametros. Si obligatoriamente necesitamos que uno de los valores sea pasado por argumento, el parametro que se use por defecto debe estar en la ultima posicion: (nombre, pais="Italia").

Tipos de funciones

Funcion Declarada

Es la funcion que se crea en el flujo del codigo principal (ambito global), con su sintaxis original, usando la palabra reservada function.

  • Hoisting: Una funcion declarada es global (visible en todo el script), cuando javascript ejecuta el codigo busca primero las funciones declaradas y las eleva (Hoisting) osea las carga en memoria antes de ejecutar el codigo y despues lo ejecuta secuencialmente, teniendo ya acceso a estas funciones, pueden ser invocada en cualquier parte del codigo..
  • Scope: las variables locales dentro de la funcion no son accesibles afuera, ya que se encuentran dentro de un bloque. Por otra parte una funcion declarada dentro de un bloque tampoco es accesible fuera de ese bloque.
  • Closures: una funcion puede "recordar" variables del entorno donde fue creada.
Expresion de Funcion

Existe otra sintaxis para crear una función conocidas como Expresiónes de Función. Estas se crean en medio de cualquier expresion de asignacion (variable = funcion), aqui la funcion se asigna como valor de una variable, tomando su nombre y permitiendo ser llamada desde el. Son funciones anonimas ya que se crean sin un nombre propio.

  • Hoisting: estas funciones son creadas cuando el flujo de ejecucion las alcanza, esto significa que no pueden ser usadas antes de su creacion, sino a partir de esta.

Un parametro
function saludar(nombre) {
  return 'Hola' + nombre + "👋 Bienvenido"
}

Dos parametros
function saludar2(nombre , apellido) {
  return 'Hola $(nombre) $(apellido) 👋 Bienvenido'
}

Esta funcion recibe un valor de entrada en el parametro "nombre" y lo usa para devolver un valor de salida con return, el resultado se muestra por pantalla llamando a la funcion saludar.

Pasar funciones como argumentos (Callback)

Un callback en javascript es simplemente una funcion que se pasa como argumento a otra funcion, y que se ejecuta despues de que ocurre algo especifico como: terminar una tarea, recibir una respuesta, o completar una accion.

JavaScript es un lenguaje asíncrono, lo que significa que puede ejecutar tareas sin bloquear el flujo principal. Los callbacks permiten que una función diga: "Cuando termines, llama a esta otra función para continuar". Esto es posible porque en JavaScript las funciones son ciudadanos de primera clase, se pueden asignar a variables, pasar como parámetros y devolver desde otras funciones.

  • Función de orden superior es la funcion que recibe otra función como argumento.
  • Callback es la función que se pasa y que será llamada dentro de la función de orden superior, ya sea de forma inmediata (síncrona) o después de que ocurra algo (asíncrona).

funcion que se pasa como callback
function saludar(nombre) {
  console.log("hola" + nombre)
}

funcion de orden superior
function procesarUsuario(callback) {
  const nombre = "Luis"
  callback(nombre) //aqui se llama el callback
}

Aqui tenemos dos funciones declaradas: procesarUsuario es de orden superior, lo que quiere decir que recibe la funcion saludar como argumento y la llama dentro de su proceso de ejecucion, pasandole como paramentro la variable nombre

por otro lado la funcion saludar es el callback de la funcion de orden superior y ejecuta un codigo con el argumento que este le dio. La palabra callback no es una palabra clave, solo es el nombre que se acostumbra para esa variable que recibira la funcion como argumento.

Anidar funciones (Nested Funcion)

Son funciones que se definen dentro de otras funciones, desde ellas podemos acceder a propiedades de cualquiera de las otras funciones, Se pueden llamar dentro de ellas mismas y acceder a sus variables.

Sintaxis

function uno() {
  function dos() {
    function tres() {
    }
  }
}

Closure

es la capacidad que tiene una funcion de tener acceso a variables de un ambito externo (osea una función interna que tiene acceso a las variables de una función externa), incluso despues de que la funcion externa que la creo haya terminado de ejecutarse, lo que quiere decir que "recuerda" el entorno o contexto en el que fue creada y guarda una pequeña memoria de las variables que estaban en su alcance cuando se creo.

Este comportamiento se debe al ámbito léxico, el cual se define en el momento en que se crea una función. Este ámbito permite que las funciones accedan a variables dentro de su ámbito y a cualquier variable de los ámbitos superiores.

function saludo() {
  let mensaje = "Hola Bienvenido/a" // ambito lexico
  let numero = 1

  function imprimir(nombre) { // closure
    console.log(mensaje + " " + nombre + " " + numero)
    numero ++
  }

return imprimir

}

const users = saludo()
users("Luis")
users("Sharon")
users("Adriana")

  • saludo() crea dos variables y define una funcion interna que luego retorna
  • imprimir() accede a las variables mensaje y numero aunque esten fuera de su propio scope, gracias al closure esta funcion retiene los datos de su ambito lexico en la que fue creada.
  • return devuelve la funcion imprimir sin los parantesis, lo que quiere decir que no se ejecuta.
  • Invocar funcion aqui "saludo" retorna la funcion interna "imprimir" pero no la ejecuta, porque no se esta invocando, para hacerlo tenemos dos maneras:
    • Ejecutar directamente saludo()(): primero "saludo" devuelve "imprimir" y luego el segundo () la ejecuta.
    • Guardarla y usarla varias veces: se guarda el resultado "imprimir" en una variable nueva y luego la llamamos, de esta manera podemos reutilizar la funcion "imprimir" con su propio ambito lexico, lo que puede requerir mas memoria si no se gestiona correctamente.
  • mostrar esta variable guarda la funcion "imprimir", junto con su entorno cerrado, es decir el closure.
  • mostrardos esta variable tambien guarda la funcion "imprimir", pero crea un nuevo closure con su propio ambito.
  • Aunque saludo() ya terminó, la función imprimir() sigue teniendo acceso a mensaje y numero porque JavaScript guarda ese contexto. Los valores de numero son independientes entre mostrar y mostrardos, proque cada closure mantiene su propio estado.

Almacenar funciones en objetos (Metodos)

Un metodo es una funcion asociada a un objeto. Es decir, se trata de una funcion que se declara como propiedad de un objeto y generalmente opera con los datos internos de ese objeto, por lo tanto un metodo es esencialmente una funcion que opera sobre propiedades o comportamientos de un objeto. Esta simple relación marca una diferencia fundamental con las funciones normales, que están "desvinculadas" de objetos concretos. El uso de métodos es especialmente relevante al trabajar con programación orientada a objetos (POO), donde los métodos representan comportamientos de los objetos.

Vale la pena enfatizar que, aunque todos los métodos son funciones, no todas las funciones son métodos. La esencialidad de un método radica en que forma parte de un objeto y, normalmente, utiliza las propiedades del mismo a través de la palabra clave this. Esta relación permite modelar conductas y operaciones relacionadas directamente con los datos que representa cada objeto.

this es la palabra clave que se usa dentro de un metodo declarado en un objeto literal para acceder a las propiedades del mismo

const persona = {
  nombre: "Sharon",
  apellido: "Gamez",
  edad: 31,
  saludar: function () { //abreviado seria: saludar () {}
    return `Hola soy $(this.nombre) $(this.apellido)
    y tengo $(this.edad) años`
  }
}

Aqui el método saludar es una funcion anonima que es una propiedad del objeto persona y tiene acceso directo a sus propiedades, como nombre, a través de this. Esta capacidad es la que distingue a los métodos y los hace potentes para definir comportamientos encapsulados

Abreviacion: los métodos pueden declararse en objetos literales usando varias sintaxis. Una forma abreviada mas simple es estableciendo el nombre de la funcion directamente sin escribir la palabra reservada function: saludar () {}.

Lamado de metodo: Para invocar el metodo, se utiliza la notacion de punto: objeto.metodo()

Almacenar funciones en clases (Metodos en clases)

Con la llegada de ECMAScript 6 (ES6), JavaScript incorporó la sintaxis de clases, facilitando la creación de estructuras orientadas a objetos. Los métodos en clases se definen de manera concisa y presentan numerosas ventajas para proyectos de cierta envergadura.

class Persona {
  constructor(nombre,hobbie) {
   this.nombre = nombre
   this.hobbie = hobbie
  }

  saludar () {
    return `Hola soy $(this.nombre)`
  }

  instrumento () {
    return `y toco la $(this.hobbie)`
  }

}

const david = new Persona ("David","guitarra")
console.log( david.saludar() + david.instrumento() )


En este ejemplo, la clase Persona tiene dos métodos: saludar e instrumento. Ambos métodos pueden operar sobre los atributos de la instancia usando this. Para utilizar los métodos, primero se crea una instancia de la clase y luego se invocan mediante la notación de punto.

Funciones Generadoras

Tambien conocida como generadores, son funciones especiales que pueden pausar su ejecucion, luego volver al punto donde se quedaron, recordando su scope y retornando valores. Se utilizan para trabajar con datos en grandes cantidades, porque ocupan poca memoria con respecto a si creamos un array u objeto.

Sintaxis

function* generador(parametros) {
  //codigo
  yield pausa la ejecucion
}

//crear generador
const valores = generador()

//llamar generador
valores.next().value
  • function* seguida de un asterisco es la palabra reservada que usamos para crear la funcion generadora.
  • yield "producir" esta palabra clave hace referencia al valor retornado cada vez que se invoque la funcion, recordando el valor anterior. En lugar de return que devuelve un valor y termina la ejecucion, yield la pausa, como si se tratara de puntos de control en la funcion que nos permiten retomarla cuando necesitemos.
  • next() es un metodo que se usa sobre la variable que contiene la funcion generadora, next la ejecuta hasta que encuentre un yield, momento en el que pausara la ejecucion y devolvera un objeto que contiene una propiedad value con cada valor de yield, y otra propiedad done con un valor booleaño true/false que indica si el generador a finalizado (true).

function* generador() {
  yield 10
  yield 20
  yield 30
}

Arrow Function

Una arrow function (función flecha) es una sintaxis más compacta para definir funciones anonimas, que son utiles en expresiones de funcion. Esta sintaxis elimina la palabra clave function y lo sustituye con una flecha (=>) que se situa entre los parámetros() y el cuerpo de la función.

No tienen su propio contexto this, no puede usarse como constructores y como se usan en expresiones de funcion, no puedes llamarlas antes de su definicion.

Sintaxis

const nombre = (parametros) => {
  return valor retornado
}

retorno implicito

las arrow function premiten el retorno implicito si el cuerpo se define en una sola linea que contenga el codigo sin llaves, lo que quiere decir que retorna el valor automaticamente sin escribir return. Si se requiere un codigo mas complejo en multilineas puedes seguir usando el retorno implicito pero se debe envolver el cuerpo de la funcion entre parentesis.

Uso de parametros

Si la funcion no tiene parametros, se indican los parentesis vacios. Si tiene un solo parametro, se puede omitir el uso de los parentesis para ese parametro. Si tienes dos o mas parametros, los parentesis seran obligatorios.

Funcion tradicional
const saludar = function(nombre) {
  return "Hola " + nombre
}

Arrow Function
const suma = (primero , segundo) => {
  return primero + segundo
}

 Arrow Function con retorno implicito
const resta = (primero , segundo) => primero - segundo
const multiplicar = numero => numero * 5
const numero = () => 45
