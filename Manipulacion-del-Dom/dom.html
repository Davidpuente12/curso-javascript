Manipulacion del DOM

El DOM (Modelo de Objetos del Documento) es una interfaz que representa la estructura de un documento HTML como un árbol de nodos, cada etiqueta html se convierte en un nodo. Estos son objetos de JavaScript que pueden ser seleccionados y manipulados, creando una interacción dinámica y personalizada entre el usuario y el contenido web.

Capacidades del DOM Descripción
🔍 Buscar etiquetas Formas y métodos para buscar elementos en el DOM como .querySelector().
🧾 Modificar contenido Acceder y modificar el contenido de una etiqueta HTML del DOM desde JavaScript.
🎭 Gestionar atributos o clases Uso de varias de las API de JavaScript para manipular clases CSS o atributos HTML.
🔮 Crear etiquetas Métodos y consejos para crear elementos en el DOM y trabajar con ellos.
🔌 Insertar etiquetas Varias de las API de JavaScript para insertar elementos en el DOM de una página.

Windows Object

El Window Object es el objeto que representa a la ventana del navegador, es decir, toda la pantalla del navegador en la que se despliega el contenido del DOM. Al estar sobre el navegador, el Window Object ofrece acceso a varias web APIs además del DOM.

Incluye una amplia gama de métodos y propiedades para interactuar con la ventana del navegador, controlar el historial, gestionar temporizadores como (setTimeout, setInterval ). Contiene sub-objetos o propiedades como history, location, navigator y uno de los más usados document.

  • Para mostrar un cuadro de alerta (alert), usamos el Window Object.
  • Si necesitamos solicitar acceso al micrófono o la cámara del usuario, se realiza a través de las APIs del Window Object.

document

es un objeto javascript que representa el arbol DOM de la pagina o mas concretamente la página web cargada en el navegador. Es tu puerta de entrada para interactuar, modificar y acceder al contenido HTML de esa página. En su interior pueden existir varios tipos de elementos, pero principalmente serán objetos de tipo Element o node.

document tambien es un nodo, mas especificamente es un nodo de tipo Document (El nodo raiz del arbol DOM) que hereda de la clase Node. Contiene el nodo <HTML> como su hijo directo. El DOM esta estructurado como una jerarquia de objetos, y todos los elementos (incluyendo Document) son instancias de clases que derivan de Node.

Elementos: (elements) es un tipo especial de nodo que representa una etiqueta HTML. Son objetos que se pueden manipular con javascript para cambiar el contenido, estilo, estructura o comportamiento.

Nodos: (node) es la unidad básica que representa cualquier parte del documento HTML o XML. Cada elemento, texto o comentario en una página web es un nodo. todo el DOM está hecho de nodos, y puedes recorrerlos, modificarlos, eliminarlos y tambien podemos agregar eventos a esos elementos.

Metodos y Propiedades de document

En el DOM de JavaScript, los métodos permiten seleccionar, modificar, recorrer y escuchar eventos en los elementos HTML.

Metodos tradicionales de busqueda de elementos

Si nos encontramos en nuestro código Javascript y queremos obtener información del contenido de nuestra página, lo primero que debemos hacer es buscar el elemento que nos interesa. La forma más sencilla de hacerlo es identificar el elemento a través de alguno de sus atributos más utilizados, generalmente el id o la clase mediante el atributo class, pero tambien existen otras formas.

getElementById()
es un metodo que pertenece a la clase Document. busca un elemento HTML con el id especificado. un documento HTML no debería tener más de un elemento con el mismo id, por lo tanto, este método devolverá siempre un solo elemento que podremos almacenar en una variable. En caso de no encontrar el elemento devuelve Null.

getElementsByClassName()
permite buscar los elementos que tengan la clase especificada en class. El método tiene getElements en plural, esto ocurre porque en este caso buscamos clases, y puede devolver varios elementos, ya que al contrario que los id, pueden existir varios elementos con la misma clase. devuelven siempre un array con todos los elementos encontrados.

getElementsByName()
funciona igual que el caso anterior. En este caso, estamos buscando elementos HTML por el valor que tienen en el atributo name, algo muy habitual en formularios HTML. Devuelven siempre un array con todos los elementos encontrados.

getElementsByTagName()
permite buscar elementos por el tipo de etiqueta HTML. Es decir, si queremos buscar todos los elementos <div> de una página, podemos hacerlo indicándole el parámetro div. devuelve un array con todos los elementos encontrados.

Metodos modernos de busqueda de elementos

En este nuevo grupo, tenemos sólo dos métodos que son equivalentes a todos los métodos tradicionales, y aún más potentes, ya que permiten hacer cosas que no se podían realizar con los anteriores (en menos codigo) gracias a CSS.

querySelector()
devuelve el primer elemento que encuentra y encaja con el selector CSS indicado por parametro. Este selector puede ser un id, una clase, un atributo name, una etiqueta (deben especificarse con su simbolo de CSS), o incluso una combinacion de selectores anidados.

querySelectorAll()
realiza una busqueda de elementos en el DOM, y devuelve una NodeList (lista de nodos) con todos los elementos que coinciden con el selector CSS especificado. A diferencia de metodos anteriores que devuelven coleccionesHTML, este devuelve NodeList que tambien pueden ser convertidos a arrays.

closest()
nos permite encontrar el primer elemento padre o ancestro mas cercano hacia arriba en el arbol DOM, que coincida con el selector CSS indicado por parametro. Es decir como un querySelector, pero que busca ascendentemente. Se usa en elementos hijos de algun otro elemento previamente seleccionado.

matches()
este metodo no sirve para buscar, pero nos permite comprobar si un elemento especifico coincide con un selector CSS indicado por parametro. Devuelve un booleano como resultado.

Estos metodos reciben por argumentos selectores de CSS
  • consultar por Id: "#nombreDeId"
  • consultar por clase: ".nombreDeClase"
  • consultar por atributo name: "[name='nombre']"
  • consultar por etiqueta: "nombreDeEtiqueta"

    Manipulacion de elementos del DOM

    Nuestra página está repleta de elementos HTML y es posible que queramos modificar uno de dichos elementos para cambiar su contenido de texto por otro. Ya sabemos localizar elementos del DOM, ahora lo que necesitamos es aprender una serie de propiedades para acceder al contenido de los elementos HTML.

    nodeName esta propiedad devuelve el nombre del nodo (la etiqueta en mayusculas). Se trata de una propiedad de solo lectura, por lo tanto no modifica el nodo, solo accede a el.

    nodeType esta propiedad devuelve el tipo de nodo (numerico: 1= elemento, 2= texto, 3= comentario). Se trata de una propiedad de solo lectura, por lo tanto no modifica el nodo, solo accede a el.

    Manipular contenido de texto

    textContent
    esta propiedad devuelve el contenido de texto plano de un elemento en concreto, sin importar su tipo. Tambien pemite cambiar el contenido de texto, por medio de el operador de asigancion (=) seguido del texto en cuestion.

    innerText
    esta propiedad devuelve el contenido de texto que se renderiza (lo que es visible para el usuario) respetando ciertos estilos css que tienen que ver con visibilidad como: display, hidden, text-transform, estos no se renderizan, por lo tanto no lo muestra. Otros estilos que no afectan el contenido textual, sino la presentacion visual (color, font-size, background) si son ignorados. Tambien permite cambiar el contenido usando el operador de asignacion (=)

    outerText
    funciona igual que innerText, teniendo en cuenta el display, de un elemento en concreto. Pero al reemplazar tambien reemplaza la etiqueta contenedora. Para cambiar el contenido usamos el operador de asignacion (=)

    Manipular contenido de HTML

    innerHTML
    nos permite obtener, reemplazar y añadir html a todo el contenido interno de un elemento. Reemplaza contenido por medio del operador de asignacion (=). Añade contenido por medio de la asignacion y suma (+=), este no es recomendable para añadir mucho contendio ya que reenderiza todo el elemento en cada insercion. Este contenido html se especifica como un string y al momento de inyectarse al DOM, los elementos se convierten a html.

    outerHTML
    similar a innerhtml, este devuelve el contenido html dede el exterior, es decir incluyendo al propio elemento contenedor. Tambien es posible modificar con el operador de asignacion.

    getHTML()
    es un metodo moderno que permite obtener el HTML de un elemento al igual que innerhtml, pero no permite modificar, solo obtener.

    insertAdjacentHTML("posicion", "HTMLString")
    tambien permite añadir contenido html pero a diferencia de innerHTML, permite especificar una posicion relativa a un elemento ya existente donde deseas insertar el html, sin reemplazar el contenido ya existente y sin renderizar todo elemento, sino solo el elemento añadido. Este metodo recibe:

      posicion: este parametro recibe palabras claves como argumento, que indican donde se inserta el htmlstring.

      • "beforebegin" Antes del elemento (afuera del nodo)
      • "afterbegin" dentro, al principio del contenido del nodo
      • "beforeend" dentro, al final del contenido del nodo
      • "afterend" despues del elemento (afuera del nodo)

      htmlstring: este parametro recibe el html que se va a insertar en forma de texto.

    createElement()
    es un metodo que permite crear elementos html usando el DOM, estos elementos son etiquetas en html. Funciona generando un nuevo nodo en el document, que luego podemos modificar, estilizar e insertar en la web. Devuelve la etiqueta que le pasemos por parametro, la cual podemos almacenar en una variable para su eventual manipulacion.

    Insertar elementos

    append() Inserta uno o varios nodos o texto como últimos hijos

    appendChild() inserta un solo nodo como ultimo hijo de un elemento. Devuelve el nodo insertado.

    prepend() añade el elemento al inicio de su contenedor

    before() inserta el elemento antes de otro elemento existente

    after() inserta el elemento despues de otro existente

    Eliminar elementos

    remove() este metodo permite eliminar de forma directa un elemento, para eso debemos identificar el elemento y usar remove sobre el, y eliminaras el elemento del DOM.

    removeChild() este metodo permite eliminar elementos de forma mas especifica, para eso debemos aplicarlo al elemento padre que contiene el elemento hijo que queremos eliminar, pasamos por parametro el elemento hijo, y eliminaras el elemento del DOM.

    Clonar elementos

    cloneNode() permite duplicar un nodo existente, manteniendo su estructura y contenido. Funciona seleccionando el elemento a clonar, luego empleamos el metodo cloneNode, almacenamos en una variable y podemos insertar en el DOM. CloneNode recibe un parametro boleano si es "true" copia los hijos del nodo, de ser false, no lo hace.

    Reemplazar elementos

    replaceWith() permite sustituir un elemento selecionado por otro, o inlcuso varios, de forma directa. Funciona en elementos, nodos de texto y comentarios. Por parametro recibe el elemento que sustituira al otro.

    Manipular Atributos HTML
    Obtener atributos

      attributes
      devuelve una coleccion (NamedNodeMap) en vivo que contiene todos los atributos de un nodo/elemento HTML. Cada atributo se representa como un objeto «Attr» con propiedades como "name" y "value". Se puede recorrer con for...of, se usa mas para inspeccion dinamica de nodos.

      hasAttributes() indica si el elemento tiene atributos HTML. Devuelve un booleano que representa la respuesta.

      hasAttribute("atributo") indica si el elemento tiene el atributo que se le indica por parametro. Devuelve un booleano que representa la respuesta.

      getAttributeNames() devuelve un array con los atributos del elemento.

      getAttribute("atributo") devuelve el valor del atributo que se le indica por parametro.

    manipular atributos
    • setAttribute("atributo", "valor")
      este metodo nos permite añadir o cambiar un atributo y su valor, si el atributo no existe lo crea, si existe le modifica el valor que indiquemos por parametro.
    • toggleAttribute("atributo", "force")
      este metodo nos permite añadir o eliminar un atributo, si el atributo no existe lo crea (auqnue no permite especificar el valor), si existe lo elimina, esto es ideal para alternar visibilidad (hidden, disabled, checked,etc).
      El parametro force es opcional, representa un booleano que verifica si el atributo esta presente en el elemento, si es true: fuerza que el atributo este presente, si es false: fuerza que el atributo se elimine, (bloquean el toggle).
    • removeAttribute("atributo")
      este metodo nos permite eliminar un atributo del elemento HTML.
    manipular y obtener

    style
    es una propiedad que permite obtener y modificar los estilos CSS en linea, de un elemento html desde el DOM, es como tener acceso al atributo (style="") de html. No elimina estilos que ya esten declarados en el html, solo añade los que se especifiquen.

    Al ser una propiedad de un objeto, usa camelCase para definir sus propiedades (background-color / backgroundColor). Si se utiliza para lectura, devuelve un [object CSSStyleDeclaration].

    className
    es una propiedad que permite obtener el valor del atributo html: class como un string. Permite reemplazar clases por medio de la asignacion, esto sobreescribe todas las clases que tenga el atributo class, si no se desea eso se debe especificar las clases existentes antes de añadir la nueva. sus metodos equivalentes serian: getAttribute("class") que devuelve el valor del atributo, setAttribute("class",value) que cambiaria el valor del atributo.

    classList
    igual a className permite obtener el valor del atributo html: class pero como un array (DOMTokenList). Al usarse la asignacion (=) por si solo tambien reemplaza todo el contenido, pero classList se diferencia ya que incorpora una serie de metodos que permiten agregar, eliminar, o alternar clases sin modificar las que ya existen.

    • item(indice) devuelve la clase indicada en el indice, null si no existe.
    • contains("clase") indica si la clase que se le pasa por parametro existe en el elemento. Devuelve un booleano que representa la afirmacion o negacion del resultado.
    • add("clase,clases") añase la clase o clases que se le pasan por parametro, si es mas de una clase se deben separar por comas.
    • remove("clase,clases") elimina la clase o clases que se le pasan por parametro, si es mas de una clase se deben separar por comas.
    • toggle("clase", condicion) permite alternar la clase que se le pasa por parametro, por medio de un booleano: true si la clase no existe, la añade, false si la clase existe, la elimina. Opcionalmente puede recibir un segundo parametro qeu sirve para establecer una condicion adicional, que de ser verdadera siempre se añadira la clase, de lo contrario siempre se eliminara.
    • replace("clase old", "clase new") reemplaza la clase que se establece en el primer parametro por la del segundo parametro.

    Eventos

    Se definen como acciones o sucesos que ocurren en el entorno de una aplicacion, desde un clic del raton hasta el envio de un formulario. Los eventos desencadenan respuestas, que permiten configurar como debe comportarse la aplicacion cada vez que ocurre algo especifico. Un evento no es más que una «notificación» de que una característica relevante acaba de suceder.

    Existen una variedad de eventos como: en el mouse (clic y desplazamiento), en el teclado (al presionar las teclas), en la ventana windows (relacionados con la carga o cambio de tamaño de las ventanas), en formularios (como entrada de datos y envio), y eventos tactiles (relevantes en dispositivos, como toques y deslizamientos).

    Existen varias formas alternativas de manejar eventos en Javascript. Vamos a ver cada una de ellas por separado, con sus particularidades:

    Manejar eventos desde HTML

    Probablemente, la forma más fácil de trabajar con eventos Javascript es mediante atributos de una etiqueta HTML, aqui los eventos tienen el perfijo «on» seguido del nombre del evento. Sin embargo, aunque es la más sencilla, también es la menos recomendable, pero es un buen punto de partida para comenzar a trabajar con eventos.

    La razon por la que es menos recomendable es que eventualmente podriamos requerir cambiar el nombre de la funcion asociada con onclick sin tener encuenta que el html depende de esas funciones, estos ocacionaria que los eventos no respondan.

    onclick
    la propiedad onclick en html y javascript se utiliza para asignar una accion que se ejecuta cuando el usuario hace clic en un elemento. Es una forma de manejar eventos de interaccion en la web. Cuando se hace clic en el elemento, ejecuta el codigo javascript que se haya definido (normalmente llamados de funciones), puede usarse directamente en el HTML o mediante Javascript.

    Sintaxis

    <button onclick="llamadoDefuncion()"> </button>

    Manejar eventos desde Javascript

    addEventListener()
    es el metodo que ofrece la capacidad de escuchar eventos que ocurren en la pagina web y permite reaccionar ante ellos. Detecta clics, teclas presionadas, movimientos del mouse, y al obtener estos eventos podemos desencadenar funciones que reaccionan a ellos. Esta funcion puede ser anonima y declararse en el mismo metodo, pero es mas recomendable pasarsela como callback.

    Sintaxis

    element.addEventListener(tipo de evento, funcionCAllback, {opciones})

    tipo de evento: este parametro recibe como argumento el nombre del evento como click, keydown, submit.

    funcionCallback: este parametro recibe como argumento la funcion que se ejecuta como respuesta al evento.

    opciones: este parametro recibe como argumento un objeto opcional que contiene configuraciones. Estos son booleanos que al ser true se activan, por defecto son "false".

    • {once: true} indica que la funcion se ejecuta solo una vez, luego elimina el listener
    • {capture:true} dispara el evento al inicio en la fase de captura(capture), en lugar de al final(bubble).
    • {passive:true} la función nunca llama a .preventDefault(), mejorando así el rendimiento.

    removeEventListener()
    sirve para eliminar un listener previamente registrado, evitando que se ejecute cuando vuelva a ocurrir el evento. Su estructura es igual a addEventListener, recibe el nombre del evento y la funcion que se va a remover, aqui es necesario que la funcion sea callback, si es anonima no se podra remover. Si el listener tenia alguna opcion tambien se debe especificar en remove.

    Comportamiento por defecto

    El comportamiento por defecto se refiere a la acción o respuesta que ocurre automáticamente cuando no se especifica una instrucción diferente. Es un concepto muy común en programación, abordaremos el tema en formularios html, generalmente los formularios intentan enviar datos a un servidor, refrescando la pagina y perdiendo cualquier informacion no guardada.

    Los botones que se ecuentran en el contexto del tag form sin especificar el atributo "type", por defecto seran del tipo "submit", al ser presionados envian datos al mismo URL usando el metodo get. Este comportamiento podemos evitarlo de varias formas, veamos como solucionarlo con Javascript.

    preventDefault()
    es un método en JavaScript que se usa para cancelar el comportamiento por defecto de un evento. Es especialmente útil cuando quieres controlar lo que ocurre al interactuar con elementos como formularios, enlaces, botones, etc. Si usas preventDefault() pero el evento no tiene comportamiento por defecto, no pasa nada. Funciona cambiando la propiedad del evento defaultPrevented a true.

    Persistencia de Datos

    Una de las limitaciones que podemos tener en Javascript cliente (navegador) es que a priori no es posible guardar de forma persistente nuestros datos, de forma que cuando recargues la página, continue conservando esos datos. Sin embargo, existe un mecanismo de navegador llamado LocalStorage que permite hacer algo similar: guardar los datos que le indiquemos en el propio navegador del usuario, de modo que posteriormente pueda recuperarlos.

    localStorage

    El (API) LocalStorage, como su propio nombre indica, es un almacén que permite guardar datos de forma local, en nuestro navegador. Esto es importante porque hay que tener en cuenta que los datos sólo serán accesibles por el usuario que los ha guardado, y en el navegador que lo ha hecho, a diferencia de una base de datos completa. Para trabajar con Local Storage, existen dos métodos.

    • localStorage Solo puede almacenar cadenas de texto (string), cada dato guardado se identifica con un nombre, que será un string. y sus valores se guardan siempre como string. Si tienes objetos o arrays, pasalos a JSON.
    • Para guardar un arreglo o un objeto de JavaScript y poder recuperarlo más tarde, debes usar JSON.stringify() antes de guardarlo, y JSON.parse() al recuperarlo
    • Los datos se guardan por dominio. Desde un dominio diferente no puedes recuperar esos datos.
    • Si utilizas otro navegador, no tendrás los datos, ya que se guardan en el navegador. Puedes verlos desde el Dev Tools, en la pestaña Applications, sección LocalStorage.
    Metodos y Propiedades de LocalStorage
    • setItem("nombre","valor") Este método se usa para guardar valores en Local Storage. Permite agregar información a la misma y se estructura con dos parámetros: una clave (nombre) y un valor. Aunque convierte automáticamente cualquier cosa que no sea una cadena de texto a un string simple, es mejor hacer la conversión con JSON.stringify() antes de guardarlo.
    • getItem("nombre") (obtener elemento) Una vez almacenada la información, este método la recupera para su uso en la aplicación. Devuelve la información asociada al dato con el nombre.
    • removeItem("nombre") (eliminar elemento) Elimina el dato guardado en "nombre".
    • length (obtener longitud) Devuelve la cantidad de elementos de datos almacenados en la página actual.
    • key(indice) Muestra la key del elemento guardado en la posición number del localStorage.
    • clear() Vacía todos los datos del almacén del localStorage para esta página. No borra las que estan almacenadas en el navegador
