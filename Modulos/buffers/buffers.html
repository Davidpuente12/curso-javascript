Buffers

En la programación, un buffer es un espacio en la memoria (RAM) destinado a almacenar temporalmente datos. Estos datos se encuentran en su forma más cruda y se expresan en bytes, en la consola, los datos se muestran en formato hexadecimal. Pueden ser manipulados directamente en la memoria, En muchos casos, son más rápidos y eficientes que las operaciones tradicionales de lectura y escritura de archivos en disco, ya que eliminan la necesidad de hacer copias y conversiones innecesarias.

Buffer

En Node.js, un Buffer es una clase incorporada que representa una secuencia fija de bytes en memoria, usado para manejar datos binarios de forma eficiente. A diferencia de los arrays, un Buffer no cambia de tamaño una vez creado.

Porque existen los Buffers? Node.js trabaja mucho con: archivos(fs), sockets TCP, streams, protocolos binarios, Para todo eso, necesitas manejar bytes directamente, no solo strings. Node lo creó porque JavaScript originalmente no podía trabajar con binarios de manera nativa.

Metodos principales
Método Parámetros Descripción breve
from() (string), (array), (buffer), (arrayBuffer) Crea un buffer a partir de un string, array, otro buffer o ArrayBuffer.
alloc() (size[, fill]) Crea un buffer inicializado en ceros.
allocUnsafe() (size) Crea un buffer sin inicializar (más rápido, pero inseguro).
byteLength() (string[, encoding]) Devuelve el número de bytes que ocupa un string.
toString() ([encoding[, start[, end]]]) Convierte el buffer a string.
slice() ([start[, end]]) Devuelve una vista del buffer sin copiar memoria.
write() (string[, offset[, length[, encoding]]]) Escribe un string dentro del buffer.
equals() (otherBuffer) Compara dos buffers.
compare() (target[, targetStart[, targetEnd[, sourceStart[, sourceEnd]]]]) Compara el orden lexicográfico entre buffers.
concat() (list[, totalLength]) Concatena múltiples buffers.
includes() (value[, byteOffset[, encoding]]) Verifica si el buffer contiene un valor.
indexOf() (value[, byteOffset[, encoding]]) Devuelve la posición de un valor dentro del buffer.

Crear un buffer (espacios en memoria vacios)
const buffer = Buffer.alloc(5)
console.log(buffer)

Crear un buffer (a partir de un string)
const string = Buffer.from('Hola')
const array = Buffer.from([1,2,3,4])
console.log(string, array)

Crear un abecedario
let abc = Buffer.alloc(26)
for(let i = 0; i < 26; i++){
  abc[i] = i + 97 //97 es el código ASCII de la letra 'a'
}
console.log(abc)
console.log(abc.toString())

streams

Un stream en Node.js es un objeto que permite leer o escribir datos de forma secuencial y eficiente, en vez de cargar todo el contenido en memoria de una sola vez. Los datos se procesan en pedazos (chunks), lo que hace a los streams ideales para archivos grandes, sockets, y cualquier operación de E/S que deba ser escalable.

Tipos principales de streams
  • Streams de Lectura: se pueden leer datos desde la fuente (por ejemplo, fs.createReadStream).
  • Streams de Escritura: se pueden escribir datos hacia un destino (por ejemplo, fs.createWriteStream).
  • Streams de doble sentido: combina lectura y escritura (por ejemplo, sockets).
  • Transform: es un duplex que transforma los datos mientras pasan (por ejemplo, compresión, cifrado). Estos tipos están implementados en la API nativa de Node.js y se usan para componer tuberías (pipe) y procesar flujos de datos.

Imagina tener un archivo enorme que deseas procesar poco a poco, en lugar de leer todo el archivo de una sola vez. Ahí es donde los streams de lectura entran en juego, permitiéndote gestionar fragmentos de información que llegan en forma de 'paquetes'.
