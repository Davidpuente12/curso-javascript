Modulos Globales

El motor de JavaScript, aquel que compila tu archivo y lo convierte en código que entiende el computador, al iniciar la compilación crea un objeto global que contiene todos los modulos globales que incluye Node.js sin necesidad de importarlos.

Los objetos globales proporciona funciones y variables propias e integradas en el lenguaje o el entorno. Dependiendo la plataforma, este objeto global tendrá un nombre diferente. En el navegador el objeto global es window, para Node.js es global, y así para cada entorno. Sin embargo, en Node.js no podrás acceder a window, ni en el navegador podrás acceder a global.

En Node.js los objetos globales son componentes integrados que permite que ciertas variables y funciones estén disponibles en toda la aplicación sin importaciones explícitas. Node.js proporciona un conjunto de objetos globales disponibles en cada módulo.

Objeto global

El objeto global proporciona un ámbito compartido para variables y funciones, similar al window object en los navegadores. Las variables o funciones definidas en este espacio estarán disponibles en cualquier parte de la aplicación sin necesidad de importarlas. Sin embargo, no es recomendable definir variables globales en este objeto para evitar problemas de contaminación global.

SetInterval y ClearInterval
(establecer intervalo) SetInterval Ejecuta funciones repetidamente en intervalos de tiempo específicos, y clearInterval detiene la ejecución.

SetTimeout y ClearTimeout
(establecer tiempo de espera) SetTimeout ejecuta una función una vez después de un retraso y ClearTimeout detiene la ejecucion.

process
Proporciona información y control sobre el proceso de ejecución de Node.js. Desde variables de entorno, información del sistema, manejar eventos del ciclo de vida del proceso. es un recurso invaluable para la gestión del entorno.

require()
es la función que se utiliza para importar módulos (archivos de JavaScript u otros recursos) dentro de tu aplicación. Es parte del sistema de módulos CommonJS, que fue el estándar por defecto en Node antes de que se popularizara import/export de ES6.

require busca el archivo o paquete en la ruta indicada, envuelve el contenido en una funcion para mantener su ambito aislado, ejecuta el codigo del modulo, y lo devuelve con export, por ultimo guarda el resultado en memoria para futuras llamadas.

Sintaxis

require( "ruta/del/modulo" )

__dirname
Devuelve el nombre del directorio actual en el que se encuentra el archivo que se está ejecutando. Es útil para generar rutas absolutas independientemente de dónde se ejecute tu script.

__filename
Devuelve la ruta completa del archivo actual, incluyendo el nombre del archivo, la diferencia de __dirname, es que incluye el nombre del archivo que se está ejecutando.

console

El objeto console en node.js es una herramienta global que proporciona metodos para imprimir mensajes en la consola, similar a la consola de los navegadores web. Proporciona métodos de registro como console.log(), console.error() y console.warn(). Se utiliza comúnmente para depurar y supervisar el comportamiento de la aplicación.

Método Función
console.log() Muestra mensajes generales en la consola.
console.error() Muestra mensajes de error en la consola (normalmente en rojo).
console.warn() Muestra advertencias en la consola (normalmente en amarillo).
console.info() Muestra información en la consola.
console.assert() Imprime un mensaje de error si la condición es falsa.
console.clear() Limpia todo el contenido de la consola.
console.count() Muestra el número de veces que se ha llamado con una etiqueta específica.
console.countReset() Reinicia el contador asociado a una etiqueta.
console.table() Muestra datos en formato de tabla.
console.group() Inicia un grupo de mensajes en la consola.
console.groupEnd() Cierra el grupo iniciado con console.group().
console.time() Inicia un temporizador con una etiqueta.
console.timeEnd() Detiene el temporizador y muestra el tiempo transcurrido.
console.dir() Muestra un objeto en forma de lista interactiva de propiedades.

Objeto globalThis

El objeto globalThis en JavaScript es una propiedad que proporciona una forma estándar y uniforme de acceder al objeto global en cualquier entorno de ejecución (navegador, Node.js, Web Workers, etc.) unificando el acceso y evitando tener que usar diferentes nombres según el entorno. globalThis contiene el valor de this global, es decir, el objeto global del contexto actual.

Modulo file system ('fs')

El módulo FS (File System) en Node.js es una API que te permitirá acceder a archivos en tu sistema operativo. Con FS, podrás leer, escribir, modificar y renombrar archivos, entre muchas otras capacidades. Este módulo es crucial cuando trabajas con cualquier aplicación que necesite interacción con archivos, como bases de datos o almacenamiento de archivos.

Cargar el módulo FS dentro de Node.js es bastante sencillo, ya que está incluido por defecto en el entorno de ejecución. Solamente necesitas requerir el módulo usando el siguiente método:

(fs).promises y (fs/promises) son una versión del módulo File System (fs) basada en promesas en lugar de callbacks. Cada operación devuelve una Promise que se resuelve cuando la acción termina o se rechaza si ocurre un error, permitiendo trabajar con el sistema de archivos usando async/await o .then()/.catch().

Sintaxis

require('fs')
require('fs').promises //Cuando necesitas mezclar callbacks y promesas en el mismo archivo
require('fs/promises') //submódulo que expone solo la API basada en promesas.

Metodos principales

readFile / readFileSync Estos metodos permiten leer un fichero, readFile es la version asincrona, esta es la mas recomendable y readFileSync es la version sincrona. Estos reciben 3 parametros:

Sintaxis

archivo.readFile( File, options, callback )

  • File Ruta del archivo a leer (puede ser relativa o absoluta).
  • options (opcional), puede ser un objeto o un string que indique la codificaicon (ej. "utf8") y otras opciones.
  • callback Funcion que se ejecuta cuando la lectura termina. Recibe dos argumentos: err: error si ocurrio alguno, data: Contendio del archivo.

writeFile / writeFileSync Estos metodos permiten escribir un fichero, writeFile es la version asincrona, esta es la mas recomendable y writeFileSync es la version sincrona. Crean un archivo nuevo si no existe, y sobreescribe el contendio si el archivo existe. Estos reciben 4 parametros:

Sintaxis

archivo.writeFile( File, Contenido, options, callback )

  • File Ruta del archivo (string, Buffer, URL o descriptor) relativa o absoluta.
  • Contenido Contenido a escribir (string, Buffer, TypedArray, DataView)
  • options (opcional), puede ser un objeto o un string que indique la codificacion (ej. "utf8"), permisos, etc.
  • callback Funcion que se ejecuta cuando la lectura termina. Y recibe error (error si ocurrio alguno).

unlink / unlinkSync se utiliza para eliminar archivos o enlaces simbólicos del sistema de archivos, no sirve para eliminar directorios. Es asincrono, pero tiene su variante sincrona unlinkSync y su variante basada en promesas (fs.promises.unlink). Recibe dos parametros:

Sintaxis

archivo.unlink( File, callback )

  • File Ruta del archivo o enlace simbólico a eliminar. (puede ser relativa o absoluta).
  • callback Funcion que se ejecuta cuando la lectura termina. Y recibe error (error si ocurrio alguno).

const file = require('fs');

file.readFile( './Modulos/archivo.txt', "utf8", (error, data) => {

if (error) {
  console.error(`Error al leer el archivo: ${error}`)
} else {
  console.log(`Contenido del archivo: ${data}`)
}
})

file.writeFile( './Modulos/nuevo.txt', 'Hola Mundo', "utf8", (error) => {

if (error) {
  console.error(`Error al escribir el archivo: ${error}`)
}
})

file.unlink( './Modulos/nuevo.txt', (error) => {

if (error) {
  console.error(`Error al borrar el archivo: ${error}`)
}
})

El modulo fs es parte de Node.js, y solo funciona en el lado del servidor.

En el navegador, no existe require ni el modulo fs, porque el navegador no tiene acceso directo al sistema de archivos por razones de seguridad.

Modulo Proceso hijo ('child_process')

Un proceso es una instancia de un programa que se está ejecutando. Por ejemplo, cada vez que ejecutamos un comando en la terminal, se crea un nuevo proceso para ejecutar dicho comando.

Node.js no solo permite ejecutar sus propios procesos, sino que también nos capacita para ejecutar otros procesos del sistema. Esto es útil en diversos casos prácticos, como cuando necesitamos interactuar con comandos del sistema operativo o trabajar con scripts externos. Descubrir cómo hacerlo amplía considerablemente nuestras posibilidades para el desarrollo con Node.js.

El módulo child_process en Node.js es un módulo integrado que permite crear y manejar procesos hijos (subprocesos) desde tu aplicación. Esto es útil porque Node.js corre en un solo hilo, y a veces necesitas ejecutar tareas externas o intensivas sin bloquear el hilo principal.

  • Ejecuta comandos del sistema desde Node.js.
  • Correr tareas intensivas en procesos separados para no bloquear el event loop.
  • Aprovechar múltiples núcleos de CPU ejecutando procesos en paralelo.
  • Comunicar tu aplicación con programas externos o scripts.
Metodos principales para crear subprocesos
Método Función
spawn() Ejecuta un comando externo y permite interactuar con su salida en tiempo real.
exec() Ejecuta comandos en una shell y devuelve toda la salida en un callback.
execFile() Ejecuta directamente un archivo binario sin pasar por un shell. Es más seguro y rápido que exec().
fork() Crea un nuevo proceso Node.js, ideal para comunicación entre procesos mediante mensajes.

exec() Sirve para ejecutar comandos del sistema operativo en un shell (por defecto /bin/sh en Unix o cmd.exe en Windows) es asincrónico y utiliza un callback para manejar la salida del comando (stdout y stderr), devuelve toda la salida de golpe, no en tiempo real. Más simple que spawn, pero menos eficiente para grandes volúmenes de datos. Recibe dos parametros:

exec('comando', (error, stdout, estderr) => { codigo } )
  • El comando que quieres ejecutar (ejemplo: "ls -lh" en Linux/macOS o "dir" en Windows).
  • Un callback que recibe: (error) si ocurre un error al ejecutar el comando, (stdout) la salida estandar del comando, y (stderr) la salida de error del comando.

spawn() Se utiliza para ejecutar comandos del sistema operativo o programas en un proceso secundario, permitiendo interactuar con su entrada y salida en tiempo real, es asincrónico y utiliza eventos para manejar la salida del proceso hijo, No usa un shell por defecto (a diferencia de exec()), a menos que lo especifiquemos con la opcion {shell: true}.

Permite streaming de datos: la salida (stdout) y los errores (stderr) se reciben en fragmentos mientras el proceso corre. Ideal para procesos largos o que generan mucha salida. Se puede enviar datos al proceso hijo mediante stdin.

spawn('comando', ['argumentos'] )
  • El comando que quieres ejecutar (ejemplo: "ls -lh" en Linux/macOS o "dir" en Windows) y los argumentos.

const { exec, spawn } = require('child_process');

exec('dir', (error, stdout, estderr) => {

if(error) {
  console.log('Error ejecutando el comando: ', error)
  return
} if(estderr) {
  console.log('Error ejecutando el comando: ', estderr)
  return
}
console.log(stdout)
})

Modulo sistema operativo 'os'

Node.js ofrece un módulo llamado 'os' que permite acceder a una serie de características del sistema operativo que usualmente estarían reservadas para lenguajes de bajo nivel. Estas funcionalidades son de gran utilidad para desarrolladores que necesitan interactuar con la memoria RAM, el número de cores de la CPU, el tipo de sistema de archivos, entre otras cosas.

Es un módulo nativo (no requiere instalación) y ofrece métodos y propiedades que permiten consultar detalles del sistema y del usuario. Para obtenerlo usaremos una variable con la siguiente sintaxis:

const os = require('os')
Método/Propiedad Qué devuelve Ejemplo práctico
arch() Arquitectura de la CPU (x64, arm, etc.) Detectar compatibilidad de binarios
platform() Plataforma del SO (linux, win32, darwin) Ejecutar código condicional según sistema
cpus() Información de cada núcleo de CPU Medir paralelismo disponible
totalmem() Memoria total del sistema (bytes) Calcular límites de caché
freemem() Memoria libre disponible Monitorear consumo de recursos
constants Constantes del sistema señas y códigos de error clave.
hostname() Nombre del host Identificar máquina en red
networkInterfaces() Interfaces de red y direcciones IP Configurar servidores o sockets
homedir() Carpeta home del usuario actual Guardar configuraciones personalizadas
uptime() Tiempo que lleva encendido el sistema (segundos) Métricas de disponibilidad

Funciones principales del modulo

  • La arquitectura de la máquina indica si estás trabajando con un sistema de 32 o 64 bits, y esto puede influir en la manera en la que compilamos o ejecutamos algunos módulos nativos. Para obtener esta información en Node.js, utilizamos arch().
  • Conocer la plataforma sobre la que tu código corre puede ser esencial, especialmente si tu software requiere ejecutar comandos específicos de un sistema operativo. El método platform() te proporciona esta información
  • Saber cuántos núcleos tiene tu procesador y cómo se están utilizando puede ayudar a optimizar el rendimiento de tus aplicaciones. Node.js te permite acceder a esta información con cpus().
  • Cuando desarrollamos aplicaciones que requieren gran cantidad de memoria, es crucial saber cuánta memoria libre hay disponible y cuál es su capacidad total. Puedes utilizar freemem() y totalmem() para esto.
  • Node.js ofrece una lista de constantes del sistema que pueden ser vitales al trabajar con procesos a muy bajo nivel. La propiedad constants te proporciona una serie de señas y códigos de error clave
  • Saber dónde almacenar archivos temporales o específicos del usuario es esencial para evitar errores o pérdida de información. Node.js nos da acceso a esta información con homedir() y tmpdir().
  • Cuando se trabaja con redes, conocer el nombre del host y las interfaces de red disponibles puede ser crítico. Node.js te proporciona los métodos hostname() y networkInterfaces() para obtener esta información.

En resumen el módulo os es como una ventana al sistema operativo: te permite saber dónde estás corriendo tu aplicación, qué recursos tienes disponibles y cómo interactuar con el entorno de manera portátil y consistente entre Windows, Linux y macOS.
