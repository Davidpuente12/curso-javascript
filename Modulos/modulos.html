Modulos

Los módulos en JavaScript son unidades de código reutilizable que permiten dividir un programa en partes independientes, facilitando la organización, el mantenimiento y la reutilización del código.

Tipos principales de módulos en JavaScript

Existen 4 tipo de módulos en JavaScript: módulos locales, módulos globales, módulos de terceros y módulos integrados.

  • Módulos Locales: Son archivos JavaScript que se crean dentro de tu proyecto y se utilizan para organizar el código.
  • Módulos Globales: Son objetos y funciones disponibles en todo el entorno de ejecución de JavaScript.
  • Módulos de Terceros: Son paquetes de código desarrollados por terceros que puedes utilizar en tu aplicación.
  • Módulos Integrados: Son módulos que vienen incluidos con el entorno de ejecución de JavaScript.

Modulos Locales

Un módulo local es un archivo de JavaScript que contiene código específico para una parte de tu aplicación. Estos módulos se crean para organizar el código, mejorar la legibilidad y facilitar el mantenimiento. Al dividir tu aplicación en módulos locales, puedes reutilizar código, evitar conflictos de nombres y mantener un código más limpio y estructurado.

Los módulos locales se crean como archivos JavaScript separados dentro de tu proyecto. Para utilizarlos, debes exportar las funciones, objetos o variables que deseas compartir con otros archivos, y luego importarlos en los archivos donde los necesites.

Sistemas de módulos en JavaScript

Existen varios sistemas de módulos en JavaScript, cada uno con sus características y usos específicos:

Tipo de módulo Contexto de uso Características clave
CommonJS (CJS) Node.js (servidor) - Usa require() y module.exports
- Carga sincrónica
- Estándar de facto en Node.js antes de ESM
AMD (Asynchronous Module Definition) Navegadores - Usa define()
- Carga asíncrona
- Diseñado para evitar bloqueos en la carga de scripts
UMD (Universal Module Definition) Compatibilidad - Combina CommonJS y AMD

Hoy en día, ESM es el estándar recomendado, ya que está soportado de forma nativa en navegadores modernos y en Node.js. CommonJS sigue siendo muy usado en proyectos Node.js antiguos o en librerías que no han migrado. AMD y UMD fueron soluciones transitorias para compatibilidad, pero hoy se usan menos.

Indicar el Tipo de módulo

Para indicar el tipo de módulo en un archivo JavaScript, puedes usar la propiedad type en el archivo package.json o especificar el tipo directamente en el archivo, en la etiqueta <script> con la propiedad y valor type="module" .

Para CommonJS, no es necesario indicar el tipo de módulo, ya que es el sistema de módulos predeterminado en Node.js. Simplemente puedes usar require() para importar módulos y module.exports para exportar funcionalidades.

Para ESM, puedes usar import y export para gestionar las dependencias.

Modulos Globales

El motor de JavaScript, aquel que compila tu archivo y lo convierte en código que entiende el computador, al iniciar la compilación crea un objeto global que contiene todos los modulos globales que incluye Node.js sin necesidad de importarlos.

Los objetos globales proporciona funciones y variables propias e integradas en el lenguaje o el entorno. Dependiendo la plataforma, este objeto global tendrá un nombre diferente. En el navegador el objeto global es window, para Node.js es global, y así para cada entorno. Sin embargo, en Node.js no podrás acceder a window, ni en el navegador podrás acceder a global.

En Node.js los objetos globales son componentes integrados que permite que ciertas variables y funciones estén disponibles en toda la aplicación sin importaciones explícitas. Node.js proporciona un conjunto de objetos globales disponibles en cada módulo.

Objeto global

El objeto global proporciona un ámbito compartido para variables y funciones, similar al window object en los navegadores. Las variables o funciones definidas en este espacio estarán disponibles en cualquier parte de la aplicación sin necesidad de importarlas. Sin embargo, no es recomendable definir variables globales en este objeto para evitar problemas de contaminación global.

Objeto globalThis

El objeto globalThis en JavaScript es una propiedad que proporciona una forma estándar y uniforme de acceder al objeto global en cualquier entorno de ejecución (navegador, Node.js, Web Workers, etc.) unificando el acceso y evitando tener que usar diferentes nombres según el entorno. globalThis contiene el valor de this global, es decir, el objeto global del contexto actual.

Exportar Modulos

Para que el código de JavaScript sea más ordenado, legible y mantenible; ES6 introduce una forma de manejar código en archivos de manera modular, de esta manera podemos desarrolar nuestro propio codigo en modulos. Esto involucra exportar funciones o variables de un archivo, e importarlas en otros archivos donde se necesite.

Modulos

Un módulo en programación es un conjunto de funciones, objetos o variables agrupadas en un archivo, que se pueden importar y utilizar en otros archivos. La modularidad mejora la gestión del código, facilita el trabajo en equipo y permite la escalabilidad de aplicaciones complejas.

Para explicar cómo funciona las exportaciones e importaciones de código, debes tener mínimo dos archivos, uno para exportar las funcionalidades y otro que las importe para ejecutarlas. Ademas debes tomar las siguientes consideraciones:

  • Si iniciaste un proyecto con NPM con Node.js, necesitas especificar que el código es modular en el archivo package.json
  • Los navegadores bloquean import si abres el HTML directamente con file://. Debes servirlo con un servidor local (ejemplo: npx serve, live-server, o el servidor integrado de VS Code).
  • Si el proyecto es sencillo, sin dependecias, debes cargar el archivo con: <script type="module">, esto cambia el comportamiento de los atributos de eventos escritos directamente en el HTML, como onclick, que dejarian de funcionar porque las funciones del modulo ya no son globales, en tal caso se deben usar eventos en el modulo.
  • Los módulos se ejecutan en modo estricto automáticamente. Las variables y funciones declaradas en un módulo no se agregan al objeto global window, pero se pueden agregar manualmente: window.variable = variable / funcion (debe tener el mismo nombre que la variable.)

export

es la instrucción que permite exportar funciones, variables, objetos o clases de un archivo para que puedan ser utilizadas en otros mediante import. Existen dos formas de exportar, antes de declarar la funcionalidad, o entre llaves {}.

Exportaciones por defecto Si solo UN valor será exportado, entonces se puede utilizar export default. De esta manera no es necesario las llaves {} al exportar e importar. Adicionalmente, no se puede usar export default antes de declaraciones const, let o var, pero puedes exportarlas al final.

import

es la instruccion para traer funciones, objetos o valores desde otro archivo o módulo al código actual, deberá estar siempre lo más arriba del archivo y utilizando el mismo nombre que el archivo original, junto a export forma el sistema de módulos de ES6.

Para importar todas las funcionalidades de un archivo se utiliza un asterisco (*) y se puede cambiar el nombre para evitar la repetición de variables o funciones a través de la palabra reservada as.

Importaciones por defecto Si únicamente un valor será importado, entonces se puede utilizar cualquier nombre en la importación. De esta manera no es necesario las llaves {}. Sin embargo, es recomendable utilizar siempre el nombre de la función, para evitar confusiones.

Importaciones dinamicas

La importación dinámica consiste en cargar los módulos cuando el usuario los vaya a utilizar, y no al iniciar la aplicación. Esto permite que la página web sea más rápida, porque descarga menos recursos. Para lograrlo utilizaremos la exprecion import() que permite manejar módulos dinámicamente, ya que la sintaxis de ECMAScript import ... from ... no lo permite.

La expresión import() es una forma especial de importar módulos de manera dinámica. A diferencia de la sentencia estática import, que se evalúa al inicio del archivo, import() es una función que devuelve una promesa y permite cargar módulos bajo demanda, en tiempo de ejecución. La expresión recibe un argumento de tipo string con la ruta del módulo a importar.

Sintaxis import { x } from "./modulo.js"; import("./modulo.js").then(...)

Retorno Exportaciones directamente Una promesa con el módulo

Modulo file system ('fs')

El módulo FS (File System) en Node.js es una API que te permitirá acceder a archivos en tu sistema operativo. Con FS, podrás leer, escribir, modificar y renombrar archivos, entre muchas otras capacidades. Este módulo es crucial cuando trabajas con cualquier aplicación que necesite interacción con archivos, como bases de datos o almacenamiento de archivos.

Cargar el módulo FS dentro de Node.js es bastante sencillo, ya que está incluido por defecto en el entorno de ejecución. Solamente necesitas requerir el módulo usando el siguiente método:

(fs).promises y (fs/promises) son una versión del módulo File System (fs) basada en promesas en lugar de callbacks. Cada operación devuelve una Promise que se resuelve cuando la acción termina o se rechaza si ocurre un error, permitiendo trabajar con el sistema de archivos usando async/await o .then()/.catch().

Sintaxis

require('fs')
require('fs').promises //Cuando necesitas mezclar callbacks y promesas en el mismo archivo
require('fs/promises') //submódulo que expone solo la API basada en promesas.

Metodos principales

readFile / readFileSync Estos metodos permiten leer un fichero, readFile es la version asincrona, esta es la mas recomendable y readFileSync es la version sincrona. Estos reciben 3 parametros:

Sintaxis

archivo.readFile( File, options, callback )

  • File Ruta del archivo a leer (puede ser relativa o absoluta).
  • options (opcional), puede ser un objeto o un string que indique la codificaicon (ej. "utf8") y otras opciones.
  • callback Funcion que se ejecuta cuando la lectura termina. Recibe dos argumentos: err: error si ocurrio alguno, data: Contendio del archivo.

writeFile / writeFileSync Estos metodos permiten escribir un fichero, writeFile es la version asincrona, esta es la mas recomendable y writeFileSync es la version sincrona. Crean un archivo nuevo si no existe, y sobreescribe el contendio si el archivo existe. Estos reciben 4 parametros:

Sintaxis

archivo.writeFile( File, Contenido, options, callback )

  • File Ruta del archivo (string, Buffer, URL o descriptor) relativa o absoluta.
  • Contenido Contenido a escribir (string, Buffer, TypedArray, DataView)
  • options (opcional), puede ser un objeto o un string que indique la codificacion (ej. "utf8"), permisos, etc.
  • callback Funcion que se ejecuta cuando la lectura termina. Y recibe error (error si ocurrio alguno).

unlink / unlinkSync se utiliza para eliminar archivos o enlaces simbólicos del sistema de archivos, no sirve para eliminar directorios. Es asincrono, pero tiene su variante sincrona unlinkSync y su variante basada en promesas (fs.promises.unlink). Recibe dos parametros:

Sintaxis

archivo.unlink( File, callback )

  • File Ruta del archivo o enlace simbólico a eliminar. (puede ser relativa o absoluta).
  • callback Funcion que se ejecuta cuando la lectura termina. Y recibe error (error si ocurrio alguno).

Modulo Proceso hijo ('child_process')

Un proceso es una instancia de un programa que se está ejecutando. Por ejemplo, cada vez que ejecutamos un comando en la terminal, se crea un nuevo proceso para ejecutar dicho comando.

Node.js no solo permite ejecutar sus propios procesos, sino que también nos capacita para ejecutar otros procesos del sistema. Esto es útil en diversos casos prácticos, como cuando necesitamos interactuar con comandos del sistema operativo o trabajar con scripts externos. Descubrir cómo hacerlo amplía considerablemente nuestras posibilidades para el desarrollo con Node.js.

El módulo child_process en Node.js es un módulo integrado que permite crear y manejar procesos hijos (subprocesos) desde tu aplicación. Esto es útil porque Node.js corre en un solo hilo, y a veces necesitas ejecutar tareas externas o intensivas sin bloquear el hilo principal.

  • Ejecuta comandos del sistema desde Node.js.
  • Correr tareas intensivas en procesos separados para no bloquear el event loop.
  • Aprovechar múltiples núcleos de CPU ejecutando procesos en paralelo.
  • Comunicar tu aplicación con programas externos o scripts.
Metodos principales para crear subprocesos
Método Función
spawn() Ejecuta un comando externo y permite interactuar con su salida en tiempo real.
exec() Ejecuta comandos en una shell y devuelve toda la salida en un callback.
execFile() Ejecuta directamente un archivo binario sin pasar por un shell. Es más seguro y rápido que exec().
fork() Crea un nuevo proceso Node.js, ideal para comunicación entre procesos mediante mensajes.

exec() Sirve para ejecutar comandos del sistema operativo en un shell (por defecto /bin/sh en Unix o cmd.exe en Windows) es asincrónico y utiliza un callback para manejar la salida del comando (stdout y stderr), devuelve toda la salida de golpe, no en tiempo real. Más simple que spawn, pero menos eficiente para grandes volúmenes de datos. Recibe dos parametros:

exec('comando', (error, stdout, estderr) => { codigo } )
  • El comando que quieres ejecutar (ejemplo: "ls -lh" en Linux/macOS o "dir" en Windows).
  • Un callback que recibe: (error) si ocurre un error al ejecutar el comando, (stdout) la salida estandar del comando, y (stderr) la salida de error del comando.

spawn() Se utiliza para ejecutar comandos del sistema operativo o programas en un proceso secundario, permitiendo interactuar con su entrada y salida en tiempo real, es asincrónico y utiliza eventos para manejar la salida del proceso hijo, No usa un shell por defecto (a diferencia de exec()), a menos que lo especifiquemos con la opcion {shell: true}.

Permite streaming de datos: la salida (stdout) y los errores (stderr) se reciben en fragmentos mientras el proceso corre. Ideal para procesos largos o que generan mucha salida. Se puede enviar datos al proceso hijo mediante stdin.

spawn('comando', ['argumentos'] )
  • El comando que quieres ejecutar (ejemplo: "ls -lh" en Linux/macOS o "dir" en Windows) y los argumentos.

console.log(stdout)

Funciones principales del modulo

  • La arquitectura de la máquina indica si estás trabajando con un sistema de 32 o 64 bits, y esto puede influir en la manera en la que compilamos o ejecutamos algunos módulos nativos. Para obtener esta información en Node.js, utilizamos arch().
  • Conocer la plataforma sobre la que tu código corre puede ser esencial, especialmente si tu software requiere ejecutar comandos específicos de un sistema operativo. El método platform() te proporciona esta información
  • Saber cuántos núcleos tiene tu procesador y cómo se están utilizando puede ayudar a optimizar el rendimiento de tus aplicaciones. Node.js te permite acceder a esta información con cpus().
  • Cuando desarrollamos aplicaciones que requieren gran cantidad de memoria, es crucial saber cuánta memoria libre hay disponible y cuál es su capacidad total. Puedes utilizar freemem() y totalmem() para esto.
  • Node.js ofrece una lista de constantes del sistema que pueden ser vitales al trabajar con procesos a muy bajo nivel. La propiedad constants te proporciona una serie de señas y códigos de error clave
  • Saber dónde almacenar archivos temporales o específicos del usuario es esencial para evitar errores o pérdida de información. Node.js nos da acceso a esta información con homedir() y tmpdir().
  • Cuando se trabaja con redes, conocer el nombre del host y las interfaces de red disponibles puede ser crítico. Node.js te proporciona los métodos hostname() y networkInterfaces() para obtener esta información.

En resumen el módulo os es como una ventana al sistema operativo: te permite saber dónde estás corriendo tu aplicación, qué recursos tienes disponibles y cómo interactuar con el entorno de manera portátil y consistente entre Windows, Linux y macOS.

Librerias de npm

Las librerías de npm (también llamadas paquetes o packages) son conjuntos de código (modulos) que otros desarrolladores han creado y publicado en el registro de npm para que puedan ser reutilizados en proyectos de Node.js o JavaScript en general.

  • npm (Node Package Manager): es el gestor de paquetes oficial de Node.js y el registro más grande de software del mundo, con millones de paquetes disponibles.
  • Paquete (package): es básicamente una carpeta o archivo comprimido que contiene código y un archivo package.json con la descripción, dependencias y metadatos del proyecto.
  • Dependencias: son librerías que tu aplicación necesita para funcionar. npm se encarga de descargarlas, instalarlas y mantenerlas actualizadas.
Diferencias entre modulos y paquetes
Concepto Funcion Ejemplo
Modulo Es cualquier archivo o conjunto de archivos de JavaScript que exporta funcionalidades (funciones, clases, objetos) y puede ser importado con require o import. modulos propios o nativos de Node.js
Paquete Es un módulo (o conjunto de módulos) empaquetado con un archivo package.json que describe su nombre, versión, dependencias y metadatos. Publicado en npm para que otros lo instalen fácilmente. express, lodash, axios

En resumen: todos los paquetes de npm son módulos, pero no todos los módulos son paquetes. Un módulo puede ser local y nunca salir de tu proyecto; un paquete es un módulo que alguien decidió empaquetar y compartir en npm.

Cifrado de contraseñas

Trabajar con contraseñas seguras es esencial en cualquier aplicación web. Una de las maneras más efectivas para hacerlo es mediante el uso del módulo bcrypt, el cual añade una capa de seguridad cifrando contraseñas y permitiendo compararlas sin exponerlas en texto plano.

El hash de contraseñas garantiza que las contraseñas de texto plano sean difíciles de encontrar para los atacantes, incluso si una base de datos está comprometida. Sin embargo, no todos los métodos de hash son iguales, y aquí es donde bcrypt destaca.

Bcrypt

Bcrypt en Node.js es una popular biblioteca de cifrado utilizada para hashear (encriptar de forma irreversible) contraseñas antes de almacenarlas en una base de datos. incorpora una "sal" (salt) automática para proteger contra ataques de tablas arcoíris y es intencionalmente lenta para dificultar ataques de fuerza bruta.

  • Hasheo Seguro: Transforma contraseñas en texto plano en cadenas irreconocibles.
  • Salting (Sal): Añade datos aleatorios únicos a cada contraseña, garantizando que contraseñas idénticas tengan hashes distintos.
  • Factor de Coste: Permite ajustar la lentitud del algoritmo para adaptarse a los aumentos de velocidad del hardware, manteniendo la seguridad a largo plazo.
  • Comparación de Contraseñas: Ofrece métodos para verificar si una contraseña ingresada coincide con el hash almacenado.

Para instalar bcrypt, necesitará tener un proyecto de Node.js configurado. Si aún no lo ha creado, siga estos pasos:

npm init -y | npm install bcrypt

Luego debes importar el modulo en tu archivo Javascript.

const bcrypt = require('bcrypt');
import bcrypt from 'bcrypt';

La librería bcrypt original está diseñada para Node.js (usa C++ interno). Si estás intentando usarla directamente en el navegador (Chrome, Firefox), te dará un error porque el navegador no entiende los módulos de Node. Prueba a usar 'bcryptjs' (una versión pura de JavaScript) que es más ligera para desarrollo: npm install bcryptjs y cambia el import a import bcrypt from 'bcryptjs';.

Metodos principales

hash() Esta función se utiliza para generar un hash de una contraseña de texto plano. Toma la contraseña de texto plano y un factor de sal (opcional) como parámetros de entrada y devuelve la contraseña hash de forma asíncrona.

bcrypt.hash( data, saltRounds, callback )
  • data es la cadena de texto que quieres hashear (normalmente la contraseña del usuario).
  • saltRounds es un número entero que indica el “coste” del algoritmo. Cuanto mayor sea, más lento será el cálculo del hash, pero también más seguro contra ataques de fuerza bruta. Valor típico: 10 (equilibrio entre seguridad y rendimiento).
  • callback (opcional en la versión asíncrona). Es una función que recibe dos parámetros: err (maneja el error) y hash (el hash generado). Se ejecuta cuando el proceso de hashing termina.

El navegador es muy estricto con los import. Solo acepta tres cosas:
Rutas relativas: ./mi-archivo.js
Rutas absolutas: /scripts/mi-archivo.js
URLs completas: https://sitio.com/libreria.js

cuando intentas usar módulos de Node.js directamente en el navegador sin un empaquetador (como Vite, Webpack o Parcel). El navegador no sabe dónde buscar la carpeta node_modules. Para él, "bcryptjs" es solo una palabra sin sentido, a menos que le des una ruta relativa (./) o una URL completa.
