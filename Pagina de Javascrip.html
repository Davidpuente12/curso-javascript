Estructuras de datos

Una vez que ya conocemos los tipos de datos elementales de Javascript, tenemos que comenzar a pensar más a lo grande. Estos tipos de datos son la naturaleza más pequeña que almacenamos en nuestro código. A medida que nuestra aplicación web va creciendo y haciéndose más grande, comenzaremos a crear estructuras cada vez más complejas.

Estas estructuras se denominan estructuras de datos y conociendo las más básicas suele ser suficiente. Sin embargo, cuantas más conozcamos y dominemos, mucho mejor.

Hash Table

Es una estructura de datos que implementa el tipo de dato abstracto, similar a un objeto o mapa. Permite almacenar datos con el concepto de "key-value". De esta manera se tiene acceso a los elementos almacenados a partir de una clave generada mediante una función hash que retorna un hash, un número que identifica la posición (bucket) donde la tabla hash localiza el valor deseado.


Linked list

las linked list (listas enlazadas), son estructuras de datos lineales formadas por nodos que no se encuentran en ubicaciones contiguas (en la memory heap) y se conectan entre si mediante enlaces. Cada nodo consta de dos partes: un valor, que puede ser cualquier tipo de dato, y un puntero (enlace) denominado next que señala a la referencia en memoria del siguiente nodo en la secuencia.

La lista tiene dos extremos: la cabeza (head) que es el primer nodo y con el cual podemos recorrer toda la lista gracias a next, esto es la variable que guarda la ubicacion del siguiente nodo, y esta en cada nodo de la lista apuntando al siguiente y la cola (tail) que es el ultimo nodo, cuyo puntero next apunta a una referencia vacia null.

El uso principal de un LINKED LIST es cuando necesitamos una estructura de datos que pueda crecer indefinidamente en la que podemos añadir o eliminar rápidamente elementos en cualquier posición, ya que los nodos se asignan el cualquier parte de la memoria heap, el uso de recursos del sistema es menor.

Recorrer Linkedlist

Para recorrer una LINKED LIST, tenemos que ir “saltando” de nodo en nodo, a través de cada uno de los enlaces. No es posible acceder a través de un índice, por fuerza hay que recorrer toda la cadena.

Por su parte, la doble LinkedList es una versión mejorada de la linked list “sencilla”, donde cada nodo contiene un enlace al nodo siguiente y al anterior. Esto permite un recorrer de forma tanto hacia adelante como hacia atrás.

Tipos de LinkedList
  • Simple Linked List (Lista Enlazada Simple). La navegación siempre es hacia adelante.
  • Doubly Linked List (Lista Enlazada Doble). La navegación puede ser hacia adelante o hacia atrás.
  • Circular Linked List (Lista Enlazada Circular). En el primer elemento tiene un puntero que apunta al elemento final y elemento final apunta al primero. Esto se traduciría en un bucle infinito, también se puede combinar con las anteriores.

Trees

Los arboles son estructuras de datos jerarquica no lineal que organiza datos en una relacion padre-hijo, similar a una estructura DOM. En un arbol el nodo principal es conocido como el nodo raiz (root), desde el cual se ramifican los datos. Cada nodo puede tener varios hijos pero un solo padre y los nodos sin hijos son conocidos como hojas.

Tipos de arbol

Las estructuras de datos de árbol se pueden clasificar según la cantidad de hijos que puede tener cada nodo.

  • Arbol Binario se caracteriza por su estructura balanceada. En este tipo de arbol, cada nodo tiene un maximo de dos nodos hijos, lo que permite mantener un equilibrio en ambas direcciones del arbol.
    • Arbol de busqueda Binario (BST) es una variacion del arbol binario diseñado para hacer busquedas de manera mas eficiente, ya que organiza los valores numericos mayores a la derecha y los menores a la izquierda del arbol.
  • Arbol Ternario cada nodo puede tener como maximo tres hijos, a menudo etiquetados como izquierdo, medio y derecho.
  • Árbol N-ario (o árbol genérico) Cada nodo puede tener cualquier número de hijos.
Eliminar nodos en un arbol de busqueda binario
  • - Nodo hoja (sin hijos) se elimina directamente, haciendo que su valor sea null

Contexto de ejecucion

Un contexto de ejecución es un entorno en el que se ejecuta el código JavaScript. Contiene información sobre el código que se está ejecutando, como las variables declaradas, las funciones y el objeto global. Cada vez que se invoca una función, se crea un nuevo contexto de ejecución.

  • Contexto global se refiere al alcanze que tienen todas las variables y codigo que esta por fuera de funciones o bloques.
  • Contexto local se refiere al alcanze que tienen las variables y codigo que estan dentro de bloques o funciones.

    • Math.random() genera un número pseudoaleatorio entre 0 (incluido) y 1 (excluido). Es decir, cada vez que lo llamas, te da un número decimal. Ejemplo: Math.random() → 0.73...

Para evitarlo: Asegúrate de modificar las variables involucradas en la condición dentro del bloque del ciclo (como por ejemplo estableciendo un contador) Esto garantiza que eventualmente el ciclo pueda salir.

    Closure

    es la capacidad que tiene una funcion de tener acceso a variables de un ambito externo (osea una función interna que tiene acceso a las variables de una función externa), incluso despues de que la funcion externa que la creo haya terminado de ejecutarse, lo que quiere decir que "recuerda" el entorno o contexto en el que fue creada y guarda una pequeña memoria de las variables que estaban en su alcance cuando se creo.

    Promesas

    Las promesas son objetos que representan el eventual resultado (completado o fallido) de una operación asincrónica. Funciona al indicar al navegador que ciertas funciones deben ejecutarse de manera independiente (como peticiones HTTP, temporizadores, lectura de archivos, etc) permitiendo trabajar de forma asincrona. Una vez resueltas, las promesas se integran en el call stack para proporcionar un resultado.

    Estados de una promesa
    • Pending (pendiente): Este es el estado inicial. La promesa está creada pero aún no se ha resuelto ni ha fallado. Cuando deja de estar pendiente, almacena un valor (si fue resuelta) o una razón (si fue rechazada).
    • Fulfilled (resuelta): Indica que la promesa se ha completado con éxito. Se gestiona a travez del then.
    • Rejected (rechazada): Ocurre cuando la promesa no se puede cumplir, produciendo un error. Se gestiona a travez del catch.

    Call Stack
    (pila de llamadas) es una pila de tareas donde se registran las funciones que deben ejecutarse. Cuando se llama a una funcion esta se agrega al callstack, ejecutandose de manera secuencial, a esto se le conoce como ejecucion sincrona.

    Operacion asincrona
    Una operación asincrónica es una tarea que se inicia y luego se ejecuta en un momento posterior, sin bloquear el hilo principal de ejecución. Esto permite que el programa continúe ejecutando otro código mientras la operación asincrónica se completa.

    Promise

    Una promesa se crea utilizando el constructor Promise(). Dentro del constructor, se pasa una función asíncrona como argumento. Esta función asíncrona, también conocida como función de resolución o rechazo, recibe dos argumentos:

      Funciones callback que usan las promesas
    • resolve una función de Javascript que se llama cuando la operación asincrónica se completa con éxito. Se le pasa un valor como parámetro, que representa el resultado de la operación.
    • reject una función de Javascript que se llama cuando la operación asincrónica falla. Se le pasa un error como parámetro, que representa la causa del fallo.

    Sintaxis

    • function(resolve, reject) esta funcion es el argumento del constructor "Promise", recibe dos parametros que representan dos funciones de Javascript. Estos callbacks seran llamados en medio de la funcion asincrona y reciben un valor que luego retornan, este valor representa el resultado de la operacion segun sea el caso: exito o error, y se mostrara en el estado de la promesa.
    • codigo asincrono aqui va la logica asincrona, y una condicion que pregunta si la condicion se cumple llamamos a resolve y si no llamamos a reject.
    • variable contiene una instancia del objeto "Promise", este objeto no guarda un valor, sino mantiene el estado y comportamiento que permiten gestionar operaciones asincronas. Solo es visible cuando se manipula con metodos como: "then", "catch", "finally".

    asincrono/esperar es un paradigma que nos permite escribir código asíncrono de forma más legible y estructurada. En JavaScript, el uso de async/await nos permite manejar promesas de una manera más clara, comparada con los métodos tradicionales que usan .then() y .catch(). Cuando declaras una función con async, esta automáticamente devuelve una promesa. Por otro lado, await hace que JavaScript espere a que una promesa se resuelva antes de continuar con el siguiente bloque de código.

    • async es una palabra clave que se coloca antes de una funcion, hace que esa funcion siempre devuelva una promesa, si la funcion devuelve un valor normal, javascript lo envuelve automaticamente en una promesa resuelta.
    • await solo puede usarse dentro de una funcion async, su funcion es pausar la ejecucion de la funcion hasta que la promesa se resuelva o rechace, haciendo que el codigo asincrono se lea como si fuera sincrono.
    • try y catch (intentar/atrapar) son la base del manejo de errores en codigo asincrono y sincrono, try maneja el bloque de codigo que puede fallar, catch maneja el codigo que fallo dentro de try, y lo guarda en una variable que definimos como parametro.

    Metodos de Peticion

    Cuando hablamos de métodos de petición (o HTTP request methods), nos referimos a las acciones que un cliente (ej. tu navegador o una app) le pide a un servidor que realice sobre un recurso. En respuesta, el servidor provee la información o realiza acciones solicitadas, lo que permite al cliente renderizar páginas, actualizar datos o recibir contenido nuevo. Los metodos de peticion son conocidos como los “verbos” del lenguaje HTTP, y cada uno tiene un propósito distinto.

    HTTP significa HyperText Transfer Protocol (Protocolo de Transferencia de Hipertexto). Es el lenguaje estándar que usan los navegadores (clientes) y los servidores para comunicarse e intercambiar información en la web. HTTPS es una versión más segura o una extensión de HTTP. En HTTPS, el navegador y el servidor establecen una conexión segura y cifrada antes de transferir datos.

    • Permite que tu navegador pida páginas, imágenes, videos, etc. a un servidor.
    • Define cómo se hacen las peticiones (con métodos como GET, POST, PUT, DELETE).
    • Define cómo se envían las respuestas (con códigos de estado como 200 OK, 404 Not Found, 500 Error).

    Funcion de las URL en las solicitudes HTTP

    Cada vez que ingresamos una URL en un navegador, como "platzi.com", se inicia un proceso en el que se envía una solicitud HTTP al servidor correspondiente. Este mecanismo es el que permite al cliente obtener la información necesaria para mostrar al usuario la página solicitada. Si todo va bien, el servidor proporciona el contenido; si hay un problema, como un recurso que no se encuentra, devuelve un error 404.

    Metodos/Verbos HTTP para peticiones

    Los verbos HTTP son esenciales para definir la acción que se desea realizar durante una interacción cliente-servidor. A continuación, exploramos los verbos más comunes:

    • GET Se utiliza para obtener información que ya existe en el servidor sin realizar ningún cambio. Por ejemplo, al acceder a "platzi.com", se envía un GET para recibir y mostrar la página en el navegador.
    • POST Este verbo es empleado para enviar datos nuevos al servidor. Cuando publicamos una actualización en redes sociales, como en Facebook, se usa un POST para almacenar esa nueva información en la base de datos del usuario.
    • PUT Se utiliza para actualizar un recurso existente, reemplazando su contenido por completo.
    • PATCH Se utiliza para actualizar un recurso existente, modificando solo los datos especificados.
    • DELETE Se emplea cuando deseamos eliminar un recurso del servidor. Puede ser la foto que subimos hace años y que ya no queremos conservar.

    fetch

    Es una función global de JavaScript que permite realizar peticiones HTTP desde el navegador de forma asíncrona y basada en Promesas; esta información la podemos manejar y mostrar en el DOM. Mediante fetch podemos hacer pedidos a una API, que a menudo nos devuelve datos en formato JSON. La Promesa representa un valor que podría estar disponible ahora, en el futuro, o nunca. Mientras la solicitud viaja por la red (tarda un tiempo), el resto de tu código JavaScript continúa ejecutándose.

    • URL en este argumento especificamos una cadena con la direccion del recurso del que queremos obtener datos.
    • opciones{} este parametro es opcional, recibe un objeto en donde podemos especificar algunas caracteristicas del fetch como:
      • method aqui se define el metodo HTTP de la peticion (como string y en mayusculas). Por defecto usa "GET", pero tambien podemos especificar los otros metodos: "POST", "PUT" etc.
      • headers cabeceras HTTP. Por defecto, se define dentro de un objeto {}.

    Metodos de fetch

    then es un metodo que sirve para manejar las promesas. Recibe una función callback donde su parámetro response es el objeto de respuesta de la petición que hemos realizado, eso quiere decir que recibe la respuesta que nos llega del servidor web al momento de recibir nuestra petición. En su interior realizaremos la lógica que queramos hacer con la respuesta a nuestra petición.

    response (variable) representa un objeto response nativo de Javascript que contiene los metadatos de la respuesta HTTP, Cuando usas fetch(url) en JavaScript, la promesa se resuelve con este objeto Response. Generalmente, el siguiente paso es llamar a un método como .json() sobre este objeto para extraer los datos que están dentro del body. Podemos acceder a los valores del objeto response como lo hariamos con otros objetos.

    ok true Un booleano que es `true` si el código de estado HTTP está en el rango 200-299 (lo que indica éxito). Es una manera rápida de verificar el éxito.

    Este código es una cadena de promesas que realiza una solicitud GET a una API de ejemplo que devuelve información ficticia.

    • fetch() recibe como argumento una URL y devuelve una Promise que se resuelve con un objeto response.
    • .then(response) se encadena a la promesa que devuelve fetch y response es el objeto que representa la respuesta HTTP. Se ejecuta solamente cuando la solicitud de red ha terminado y la promesa se ha resuelto (es decir, el servidor a respondido)
    • json() es un metodo clave de objeto response, este lee el cuerpo de la respuesta de forma asincrona (lee response.body ), convierte el cuerpo de la respuesta en un objeto javascript, y retorna una nueva Promise, que se resuelve con otro objeto final (data).

      • ¿Porque devuelve una promesa?

      El método response.json() devuelve una Promesa porque el proceso de leer el cuerpo (body) de la respuesta HTTP y convertirlo de una cadena de texto a un objeto JavaScript utilizable (JSON) es una operación asíncrona que toma tiempo. ⏳

      • 1. El Cuerpo es un "Stream" (Flujo de Datos)

        Cuando recibes una respuesta de red, el cuerpo de la respuesta (response.body) no es un simple objeto ya listo, sino un ReadableStream. Esto significa que los datos pueden llegar en partes a lo largo del tiempo, especialmente si el archivo es grande.
      • 2. Lectura y Conversión

        El método response.json() tiene que realizar dos tareas importantes: Leer todo el stream de datos hasta que el cuerpo esté completo. Analizar (parsear) la cadena de texto completa para convertirla en el objeto JavaScript (JSON) que necesitas.
      • 3. Evitar el Bloqueo del Hilo Principal

        Si estas operaciones (leer el stream y parsear) se ejecutaran de forma síncrona, el hilo principal de tu navegador (donde se ejecuta el JavaScript que maneja la interfaz de usuario) se bloquearía completamente. Esto haría que la página se congelara y no respondiera hasta que se completara el proceso.

        Al devolver una Promesa, response.json() permite que la lectura y el parseo ocurran en segundo plano sin congelar la interfaz. La Promesa simplemente indica: "Te notificaré (a través del siguiente .then()) cuando el trabajo esté hecho y los datos estén listos." De esta forma, el flujo de fetch es completamente asíncrono:

        fetch(url) → Devuelve una Promesa (por la solicitud de red).
        response.json() → Devuelve una Promesa (por la lectura y conversión de datos).
    • .then(datos) este segundo then se encadena para esperar a que la Promesa devuelta por response.json() se resuelva: espera a que el cuerpo de la respuesta haya sido leido y convertido a un objeto Javascript(el paso anterior), datos es el objeto javascript que representa los datos JSON extraidos y lo muestra por consola.

    Metodos tradicionales

    es un metodo que pertenece a la clase Document. busca un elemento HTML con el id especificado. un documento HTML no debería tener más de un elemento con el mismo id, por lo tanto, este método devolverá siempre un solo elemento que podremos almacenar en una variable.     permite buscar los elementos que tengan la clase especificada en class. El método tiene getElements en plural, esto ocurre porque en este caso buscamos clases, y puede devolver varios elementos, ya que al contrario que los id, pueden existir varios elementos con la misma clase. devuelven siempre un array con todos los elementos encontrados.

    getElementsByName()
    funciona igual que el caso anterior. En este caso, estamos buscando elementos HTML por el valor que tienen en el atributo name, algo muy habitual en formularios HTML. Devuelven siempre un array con todos los elementos encontrados.

    getElementsByTagName()
    permite buscar elementos por el tipo de etiqueta HTML. Es decir, si queremos buscar todos los elementos <div> de una página, podemos hacerlo indicándole el parámetro div. devuelve un array con todos los elementos encontrados.

    Persistencia de Datos

    Una de las limitaciones que podemos tener en Javascript cliente (navegador) es que a priori no es posible guardar de forma persistente nuestros datos, de forma que cuando recargues la página, continue conservando esos datos. Sin embargo, existe un mecanismo de navegador llamado LocalStorage que permite hacer algo similar: guardar los datos que le indiquemos en el propio navegador del usuario, de modo que posteriormente pueda recuperarlos.

    localStorage

    El (API) LocalStorage, como su propio nombre indica, es un almacén que permite guardar datos de forma local, en nuestro navegador. Esto es importante porque hay que tener en cuenta que los datos sólo serán accesibles por el usuario que los ha guardado, y en el navegador que lo ha hecho, a diferencia de una base de datos completa. Para trabajar con Local Storage, existen dos métodos.

    • localStorage Solo puede almacenar cadenas de texto (string), cada dato guardado se identifica con un nombre, que será un string. y sus valores se guardan siempre como string. Si tienes objetos o arrays, pasalos a JSON.
    • Para guardar un arreglo o un objeto de JavaScript y poder recuperarlo más tarde, debes usar JSON.stringify() antes de guardarlo, y JSON.parse() al recuperarlo
    • Los datos se guardan por dominio. Desde un dominio diferente no puedes recuperar esos datos.
    • Si utilizas otro navegador, no tendrás los datos, ya que se guardan en el navegador. Puedes verlos desde el Dev Tools, en la pestaña Applications, sección LocalStorage.
    Metodos y Propiedades de LocalStorage
    • setItem("nombre","valor") Este método se usa para guardar valores en Local Storage. Permite agregar información a la misma y se estructura con dos parámetros: una clave (nombre) y un valor. Aunque convierte automáticamente cualquier cosa que no sea una cadena de texto a un string simple, es mejor hacer la conversión con JSON.stringify() antes de guardarlo.
    • getItem("nombre") (obtener elemento) Una vez almacenada la información, este método la recupera para su uso en la aplicación. Devuelve la información asociada al dato con el nombre.
    • removeItem("nombre") (eliminar elemento) Elimina el dato guardado en "nombre".
    • length (obtener longitud) Devuelve la cantidad de elementos de datos almacenados en la página actual.
    • key(indice) Muestra la key del elemento guardado en la posición number del localStorage.
    • clear() Vacía todos los datos del almacén del localStorage para esta página. No borra las que estan almacenadas en el navegador

    Mensajes Emergentes

    Javascript ofrece entre sus funciones nativas la posibilidad de crear mensajes emergentes. Más específicamente, alerta (alert), confirmación (confirm) y mensajes personalizados (Prompt)

    1. Crear un mensaje de alerta con alert()

    La función alert() nos permite desde JavaScript crear un mensaje de alerta sin algo más que un botón para aceptar, de modo que el mensaje desaparecerá cuando el usuario presione aceptar o cierre la ventana. Ejemplo: alert("mensaje de alerta");

    2. Crear un mensaje de confirmación con confirm()

    A diferencia de la función Alert, la función confirm() además de mostrar un mensaje emergente nos permite aceptar o denegar dicho mensaje. para recojer la inmformacion del usuario debemos guardarla en una variable, igualandola a "confirm()" Ejemplo:
    let b = confirm("Estas seguro de borrar?")
    alert(b)

    3. Crear un mensaje personalizado con Prompt()

    La función prompt() nos permite enviar un mensaje emergente con una caja de texto de modo que el usuario puede ingresar un valor dentro de ella. En caso de presionar «aceptar» el valor de establecerá, en caso de presionar «cancelar» en valor quedará vacío. Ejemplo: prompt("introduce tu edad") si queremos recojer la informacion que el usuario escribio en el "prompt" debemos guardarla en una variable, si queremos asignar un valor por defecto lo escribimos despues de una coma.
