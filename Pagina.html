JavaScript

JavaScript es un lenguaje de programación que permite hacer que las páginas web sean interactivas, dinámicas y funcionales. Es uno de los tres pilares del desarrollo web junto con HTML y CSS de cualquier página web, los programas creados en este lenguaje se llaman scripts. Estos scripts son texto y no necesitan ser compilados de ninguna forma para funcionar, como otros lenguajes como puede ser Java.

  • En un inicio se penso para ejecutarse en navegadores, aunque hoy en dia va mucho mas alla.
  • Cualquier dispositivo con un "JavaScript engine" o motor de JavaScript puede ejecutarlo.
  • Los navegadores incluyen un JavaScript engine.

Motor de JavaScript

tambien llamado JavaScript engine es el componente de software que ejecuta el código JavaScript. Es como el “cerebro” que traduce lo que escribes en JS a instrucciones que la computadora realmente puede entender y ejecutar. Existen varios motores de JavaScript: V8 = Chrome y Opera ; SpiderMonkey = Firefox , etc. Además, motores como V8 también se usan fuera del navegador, por ejemplo en Node.js y Deno, para ejecutar JavaScript en servidores o entornos locales.

Capacidades de JavaScript en un navegador

con JavaScript en el navegador no tenemos acceso a la memoria o CPU del sistema. aunque con JavaScript usando NodeJS si que tenemos acceso a estas cosas. En el navegador podemos hacer todo lo relacionado a la manipulacion e interaccion con el usuario de la pagina web. Muchos de estos limites dejan de existir cuando ejecutamos JavaScript fuera del navegador, como por ejemplo en el servidor.

  • Se puede hacer:
  • Añadir nuevo HTML a la web, cambiando el contenido, modificando estilos.
  • Reaccionar a las acciones del usuario, clicks, movimientos, teclas pulsadas, etc.
  • Enviar peticiones a servidores remotos, descargar y subir archivos.
  • Escribir y leer cookies, hacer preguntar al usuario, mostrar mensajes.
  • Guardar datos en el lado del cliente para recordar mas tarde.
  • Crear animaciones, juegos, y efectos visuales. Controlar audio, video y otros elementos multimedia.
  • Manipular el DOM (Document Object Model) para cambiar el contenido de la página en tiempo real.
  • No se puede hacer:
  • No podemos escribir o leer ficheros del disco duro
  • No podemos ejecutar programas instalados en el ordenador
  • Podemos interactuar con la camara o microfono del ordenador, pero siempre con confirmacion por parte del usuraio.
  • Las diferentes pestañas no saben de la existencia de otras pestañas. Excepto cuando desde una pestaña de JavaScript abre otra, pero incluso asi tendria que ser del mismo dominio, protocolo y puerto, es decir si desde tu pagina abres una pestaña de Google por ejemplo, no vas a tener acceso, tiene que ser de tu misma web.

Contexto de ejecucion

Un contexto de ejecución es un entorno en el que se ejecuta el código JavaScript. Contiene información sobre el código que se está ejecutando, como las variables declaradas, las funciones y el objeto global. Cada vez que se invoca una función, se crea un nuevo contexto de ejecución.

  • Contexto global se refiere al alcanze que tienen todas las variables y codigo que esta por fuera de funciones o bloques.
  • Contexto local se refiere al alcanze que tienen las variables y codigo que estan dentro de bloques o funciones.

scope chain

La "cadena de alcance" es el mecanismo que utiliza JavaScript para determinar qué variables están disponibles para una instrucción en particular. Se busca en una secuencia de contextos de ejecución hasta encontrar la variable con el nombre correspondiente. Un contexto local puede acceder a variables en contexto global, pero no al revés. La búsqueda de variables funciona de adentro hacia afuera, asegurándose de que no haya acceso entre contextos locales iguales.

let mensajeGlobal = "Hola desde afuera"

function mensajes() {
  let mensajeBloque = "Hola desde el bloque"
  var mensajeVar = "Hola soy var"
  console.log(mensajeGlobal) // obtiene mensajeGlobal globalmente
  console.log(mensajeBloque) // obtiene mensajeBloque localmente
  console.log(mensajeVar) // obtiene mensajeVar localmente

  • Contexto local: Desde adentro del bloque estan disponibles las variables locales como: mensajeBloque y mensajevar, pero ademas tiene acceso a variables fuera del bloque, osea a las globales como: mensaGlobal.
  • Contexto global: Desde afuera del bloque estan disponible las variables Globales como: mensajeGlobal pero no se tiene acceso a la variable dentro del bloque como mensajeBloque, y al no encontrarla se detendria el flujo de trabajo, ocacionando que no se ejecute el codigo posterior. En este caso lo mandamos a imprimir de ultimo para que no detenga el resto del codigo.

Variables de entorno

Las variables de entorno son valores dinámicos que existen fuera del código de una aplicación y que afectan cómo se ejecutan los programas, proporcionan información configurable como rutas de archivos, configuraciones de conexión, o modos de aplicación, permitiendo adaptar el comportamiento de una app sin modificar su código fuente.

Es crucial para entornos distintos (desarrollo, producción) y para proteger datos sensibles como contraseñas. Por ejemplo, en nuestro entorno de desarrollo conectamos a una base de datos de prueba, mientras que en producción nos conectamos a una base de datos real, en lugar de utilizar la dirección de la base de datos directamente, utilizaremos una variable que tendrá la dirección de la base de datos.

Estas variables se suelen guardar en un archivo llamado .env y en ella se suelen guardar datos que varían dependiendo del entorno, así como nombres de usuario o contraseñas, que son también sensibles a cambios frecuentes y no queremos que estén acoplados al código fuente.

Creando variables de entorno

Como hemos dicho, el primer paso es crear un fichero .env en nuestro raíz del proyecto. En su interior crearemos las variables en mayúsculas (separadas por guión bajo si se necesita) seguido de un = con el valor correspondiente.

Para leer variables de entorno desde NodeJS, podemos utilizar dos formas diferentes. El método loadEnvFile() de node:process o ejecutar el parámetro --env-file desde la línea de comandos. En ambos casos, le pasamos la ruta del fichero .env que contiene las variables de entorno que queremos leer en nuestra aplicación.

Manipulacion de numeros

En JavaScript, el tipo de dato primitivo number es esencial para el manejo de operaciones numéricas. Aunque existen diversas formas de escribir números, como enteros y decimales, todos se reconocen bajo el mismo tipo de dato: number. Esta unificación facilita la manipulación y el cálculo numérico. Veamos cómo se representan en este lenguaje:

  • Operadores aritmeticos: suma (+) resta(-) multiplicacion(*) divicion(/) modulo(%) exponente(**)
  • metodo toFixed() se usa para formatear un numero con una cantidad especifica de decimales. devolviendo el resultado como una cadena de texto. El numero se redondea si es necesario.
  • Objeto math da acceso a funciones y constantes matematicas.
    • Math.PI es una propiedad del objeto global Math que representa el valor de π (pi) → 3.14159...
    • Math.round(x) es una función que redondea un número al entero más cercano. Si el decimal es 0.5 o mayor, redondea hacia arriba. Si el decimal es menor que 0.5, redondea hacia abajo. ejemplo: Math.round(4.6) → 5
    • Math.floor(x) sirve para redondear un número hacia abajo al entero más cercano. Ejemplo: Math.floor(4.6) → 4
    • Math.ceil(x) es una función que redondea un número hacia arriba al entero más cercano. Ejemplo: Math.ceil(4.2) → 5
    • Math.abs(x) devuelve el valor absoluto de x, es decir, la distancia entre ese número y el cero, sin importar el signo. - Si x es positivo o cero → lo devuelve tal cual.- Si x es negativo → lo convierte en positivo. Ejemplo: Math.abs(-7) → 7
    • Math.pow(Exponente, Potencia) sirve para elevar un número a una potencia, es decir, calcular la base elevada al exponente Potencia (x elevado a y). Alternativa moderna: (**) Ejemplo: Math.pow(2, 3) → 8
    • Math.sqrt(x) devuelve la raíz cuadrada de x, es decir, el número positivo que, al multiplicarse por sí mismo, da como resultado x. Ejemplo: Math.sqrt(9) → 3
    • Math.max(valor1, valor2...) sirve para encontrar el mayor valor entre dos o más números. Es una función muy útil cuando necesitas comparar valores y quedarte con el más grande. Ejemplo: Math.max(1, 5, 3) → 5
    • Math.min(valor1, valor2...) sirve para encontrar el menor valor entre dos o más números. Ejemplo: Math.min(1, 5, 3) → 1
    • Math.random() genera un número pseudoaleatorio entre 0 (incluido) y 1 (excluido). Es decir, cada vez que lo llamas, te da un número decimal. Ejemplo: Math.random() → 0.73...

    • Hoisting: Una funcion declarada es global (visible en todo el script), cuando javascript ejecuta el codigo busca primero las funciones declaradas y las eleva (Hoisting) osea las carga en memoria antes de ejecutar el codigo y despues lo ejecuta secuencialmente, teniendo ya acceso a estas funciones, pueden ser invocada en cualquier parte del codigo..
    • Scope: las variables locales dentro de la funcion no son accesibles afuera, ya que se encuentran dentro de un bloque. Por otra parte una funcion declarada dentro de un bloque tampoco es accesible fuera de ese bloque.
    • Closures: una funcion puede "recordar" variables del entorno donde fue creada.
    Expresion de Funcion

    Existe otra sintaxis para crear una función conocidas como Expresiónes de Función. Estas se crean en medio de cualquier expresion de asignacion (variable = funcion), aqui la funcion se asigna como valor de una variable, tomando su nombre y permitiendo ser llamada desde el. Son funciones anonimas ya que se crean sin un nombre propio.

    • Hoisting: estas funciones son creadas cuando el flujo de ejecucion las alcanza, esto significa que no pueden ser usadas antes de su creacion, sino a partir de esta.
    Parametros por Defecto

    Pasar funciones como argumentos (Callback)

    Un callback en javascript es simplemente una funcion que se pasa como argumento a otra funcion, y que se ejecuta despues de que ocurre algo especifico como: terminar una tarea, recibir una respuesta, o completar una accion.

    JavaScript es un lenguaje asíncrono, lo que significa que puede ejecutar tareas sin bloquear el flujo principal. Los callbacks permiten que una función diga: "Cuando termines, llama a esta otra función para continuar". Esto es posible porque en JavaScript las funciones son ciudadanos de primera clase, se pueden asignar a variables, pasar como parámetros y devolver desde otras funciones.

    • Función de orden superior es la funcion que recibe otra función como argumento.
    • Callback es la función que se pasa y que será llamada dentro de la función de orden superior, ya sea de forma inmediata (síncrona) o después de que ocurra algo (asíncrona).

    funcion que se pasa como callback
    function saludar(nombre) {
      console.log("hola" + nombre)
    }

    funcion de orden superior
    function procesarUsuario(callback) {
      const nombre = "Luis"
      callback(nombre) //aqui se llama el callback
    }

    Aqui tenemos dos funciones declaradas: procesarUsuario es de orden superior, lo que quiere decir que recibe la funcion saludar como argumento y la llama dentro de su proceso de ejecucion, pasandole como paramentro la variable nombre

    por otro lado la funcion saludar es el callback de la funcion de orden superior y ejecuta un codigo con el argumento que este le dio. La palabra callback no es una palabra clave, solo es el nombre que se acostumbra para esa variable que recibira la funcion como argumento.

    Closure

    es la capacidad que tiene una funcion de tener acceso a variables de un ambito externo (osea una función interna que tiene acceso a las variables de una función externa), incluso despues de que la funcion externa que la creo haya terminado de ejecutarse, lo que quiere decir que "recuerda" el entorno o contexto en el que fue creada y guarda una pequeña memoria de las variables que estaban en su alcance cuando se creo.

    Este comportamiento se debe al ámbito léxico, el cual se define en el momento en que se crea una función. Este ámbito permite que las funciones accedan a variables dentro de su ámbito y a cualquier variable de los ámbitos superiores.

    function saludo() {
      let mensaje = "Hola Bienvenido/a" // ambito lexico
      let numero = 1

      function imprimir(nombre) { // closure
        console.log(mensaje + " " + nombre + " #" + numero)
        numero ++
      }
      return imprimir
    }

    const mostrar = saludo()
    mostrar("Luis")
    mostrar("Puente")

    const mostrardos = saludo()
    mostrardos("Sharon")
    mostrardos("Gamez")
    mostrar
    mostrardos
    • saludo() crea dos variables y define una funcion interna que luego retorna
    • imprimir() accede a las variables mensaje y numero aunque esten fuera de su propio scope, gracias al closure esta funcion retiene los datos de su ambito lexico en la que fue creada.
    • return devuelve la funcion imprimir sin los parantesis, lo que quiere decir que no se ejecuta.
    • Invocar funcion aqui "saludo" retorna la funcion interna "imprimir" pero no la ejecuta, porque no se esta invocando, para hacerlo tenemos dos maneras:
      • Ejecutar directamente saludo()(): primero "saludo" devuelve "imprimir" y luego el segundo () la ejecuta.
      • Guardarla y usarla varias veces: se guarda el resultado "imprimir" en una variable nueva y luego la llamamos, de esta manera podemos reutilizar la funcion "imprimir" con su propio ambito lexico, lo que puede requerir mas memoria si no se gestiona correctamente.
    • mostrar esta variable guarda la funcion "imprimir", junto con su entorno cerrado, es decir el closure.
    • mostrardos esta variable tambien guarda la funcion "imprimir", pero crea un nuevo closure con su propio ambito.
    • Aunque saludo() ya terminó, la función imprimir() sigue teniendo acceso a mensaje y numero porque JavaScript guarda ese contexto. Los valores de numero son independientes entre mostrar y mostrardos, proque cada closure mantiene su propio estado.

    5. Almacenar funciones en objetos (Metodos)

    Un metodo es una funcion asociada a un objeto. Es decir, se trata de una funcion que se declara como propiedad de un objeto y generalmente opera con los datos internos de ese objeto, por lo tanto un metodo es esencialmente una funcion que opera sobre propiedades o comportamientos de un objeto. Esta simple relación marca una diferencia fundamental con las funciones normales, que están "desvinculadas" de objetos concretos. El uso de métodos es especialmente relevante al trabajar con programación orientada a objetos (POO), donde los métodos representan comportamientos de los objetos.

    Vale la pena enfatizar que, aunque todos los métodos son funciones, no todas las funciones son métodos. La esencialidad de un método radica en que forma parte de un objeto y, normalmente, utiliza las propiedades del mismo a través de la palabra clave this. Esta relación permite modelar conductas y operaciones relacionadas directamente con los datos que representa cada objeto.

    this es la palabra clave que se usa dentro de un metodo declarado en un objeto literal para acceder a las propiedades del mismo

    const persona = {
      nombre: "Sharon",
      apellido: "Gamez",
      edad: 31,
      saludar: function () { //abreviado seria: saludar () {}
        return `Hola soy $(this.nombre) $(this.apellido)
        y tengo $(this.edad) años`
      }
    }

    Aqui el método saludar es una funcion anonima que es una propiedad del objeto persona y tiene acceso directo a sus propiedades, como nombre, a través de this. Esta capacidad es la que distingue a los métodos y los hace potentes para definir comportamientos encapsulados

    Abreviacion: los métodos pueden declararse en objetos literales usando varias sintaxis. Una forma abreviada mas simple es estableciendo el nombre de la funcion directamente sin escribir la palabra reservada function: saludar () {}.

    Lamado de metodo: Para invocar el metodo, se utiliza la notacion de punto: objeto.metodo()

    No tienen su propio contexto this, no puede usarse como constructores y como se usan en expresiones de funcion, no puedes llamarlas antes de su definicion.

    Exportar Modulos

    Para que el código de JavaScript sea más ordenado, legible y mantenible; ES6 introduce una forma de manejar código en archivos de manera modular. Esto involucra exportar funciones o variables de un archivo, e importarlas en otros archivos donde se necesite.

    Para explicar cómo funciona las exportaciones e importaciones de código, debes tener mínimo dos archivos, uno para exportar las funcionalidades y otro que las importe para ejecutarlas. Además, si iniciaste un proyecto con NPM (Node Package Manager) con Node.js, necesitas especificar que el código es modular en el archivo package.json

    export

    es la instrucción que permite exportar funciones, variables, objetos o clases de un archivo para que puedan ser utilizadas en otros mediante import. Existen dos formas de exportar, antes de declarar la funcionalidad, o entre llaves {}.

    Exportaciones por defecto Si solo UN valor será exportado, entonces se puede utilizar export default. De esta manera no es necesario las llaves {} al exportar e importar. Adicionalmente, no se puede usar export default antes de declaraciones const, let o var, pero puedes exportarlas al final.

    import

    es la instruccion para traer funciones, objetos o valores desde otro archivo o módulo al código actual, deberá estar siempre lo más arriba del archivo y utilizando el mismo nombre que el archivo original, junto a export forma el sistema de módulos de ES6.

    Para importar todas las funcionalidades de un archivo se utiliza un asterisco (*) y se puede cambiar el nombre para evitar la repetición de variables o funciones a través de la palabra reservada as.

    Importaciones por defecto Si únicamente un valor será importado, entonces se puede utilizar cualquier nombre en la importación. De esta manera no es necesario las llaves {}. Sin embargo, es recomendable utilizar siempre el nombre de la función, para evitar confusiones.

    Importaciones dinamicas

    La importación dinámica consiste en cargar los módulos cuando el usuario los vaya a utilizar, y no al iniciar la aplicación. Esto permite que la página web sea más rápida, porque descarga menos recursos. Para lograrlo utilizaremos la exprecion import() que permite manejar módulos dinámicamente, ya que la sintaxis de ECMAScript import ... from ... no lo permite.

    La expresión import() es una forma especial de importar módulos de manera dinámica. A diferencia de la sentencia estática import, que se evalúa al inicio del archivo, import() es una función que devuelve una promesa y permite cargar módulos bajo demanda, en tiempo de ejecución. La expresión recibe un argumento de tipo string con la ruta del módulo a importar.

    Mensajes Emergentes

    Javascript ofrece entre sus funciones nativas la posibilidad de crear mensajes emergentes. Más específicamente, alerta (alert), confirmación (confirm) y mensajes personalizados (Prompt)

    1. Crear un mensaje de alerta con alert()

    La función alert() nos permite desde JavaScript crear un mensaje de alerta sin algo más que un botón para aceptar, de modo que el mensaje desaparecerá cuando el usuario presione aceptar o cierre la ventana. Ejemplo: alert("mensaje de alerta");

    2. Crear un mensaje de confirmación con confirm()

    A diferencia de la función Alert, la función confirm() además de mostrar un mensaje emergente nos permite aceptar o denegar dicho mensaje. para recojer la inmformacion del usuario debemos guardarla en una variable, igualandola a "confirm()" Ejemplo:
    let b = confirm("Estas seguro de borrar?")
    alert(b)

    3. Crear un mensaje personalizado con Prompt()

    La función prompt() nos permite enviar un mensaje emergente con una caja de texto de modo que el usuario puede ingresar un valor dentro de ella. En caso de presionar «aceptar» el valor de establecerá, en caso de presionar «cancelar» en valor quedará vacío. Ejemplo: prompt("introduce tu edad") si queremos recojer la informacion que el usuario escribio en el "prompt" debemos guardarla en una variable, si queremos asignar un valor por defecto lo escribimos despues de una coma.
