JavaScript

JavaScript es un lenguaje de programación que permite hacer que las páginas web sean interactivas, dinámicas y funcionales. Es uno de los tres pilares del desarrollo web junto con HTML y CSS de cualquier página web, los programas creados en este lenguaje se llaman scripts. Estos scripts son texto y no necesitan ser compilados de ninguna forma para funcionar, como otros lenguajes como puede ser Java.

  • En un inicio se penso para ejecutarse en navegadores, aunque hoy en dia va mucho mas alla.
  • Cualquier dispositivo con un "JavaScript engine" o motor de JavaScript puede ejecutarlo.
  • Los navegadores incluyen un JavaScript engine.

Motor de JavaScript

tambien llamado JavaScript engine es el componente de software que ejecuta el código JavaScript. Es como el “cerebro” que traduce lo que escribes en JS a instrucciones que la computadora realmente puede entender y ejecutar. Existen varios motores de JavaScript: V8 = Chrome y Opera ; SpiderMonkey = Firefox , etc. Además, motores como V8 también se usan fuera del navegador, por ejemplo en Node.js y Deno, para ejecutar JavaScript en servidores o entornos locales.

Capacidades de JavaScript en un navegador

con JavaScript en el navegador no tenemos acceso a la memoria o CPU del sistema. aunque con JavaScript usando NodeJS si que tenemos acceso a estas cosas. En el navegador podemos hacer todo lo relacionado a la manipulacion e interaccion con el usuario de la pagina web. Muchos de estos limites dejan de existir cuando ejecutamos JavaScript fuera del navegador, como por ejemplo en el servidor.

  • Se puede hacer:
  • Añadir nuevo HTML a la web, cambiando el contenido, modificando estilos.
  • Reaccionar a las acciones del usuario, clicks, movimientos, teclas pulsadas, etc.
  • Enviar peticiones a servidores remotos, descargar y subir archivos.
  • Escribir y leer cookies, hacer preguntar al usuario, mostrar mensajes.
  • Guardar datos en el lado del cliente para recordar mas tarde.
  • Crear animaciones, juegos, y efectos visuales. Controlar audio, video y otros elementos multimedia.
  • Manipular el DOM (Document Object Model) para cambiar el contenido de la página en tiempo real.
  • No se puede hacer:
  • No podemos escribir o leer ficheros del disco duro
  • No podemos ejecutar programas instalados en el ordenador
  • Podemos interactuar con la camara o microfono del ordenador, pero siempre con confirmacion por parte del usuraio.
  • Las diferentes pestañas no saben de la existencia de otras pestañas. Excepto cuando desde una pestaña de JavaScript abre otra, pero incluso asi tendria que ser del mismo dominio, protocolo y puerto, es decir si desde tu pagina abres una pestaña de Google por ejemplo, no vas a tener acceso, tiene que ser de tu misma web.

Declaraciones

En JavaScript las intrucciones de denominan "declaraciones" estas le dicen al interprete que realice una accion especifica. Es como una linea de ordenes que el navegador ejecuta paso a paso. Cada declaracion puede: crear una variable, definir una funcion, realizar una condicion, iniciar un bucle.

  • Punto y coma (;) este simbolo se usa separar las declaraciones, indicando que hasta alli llega su ejecucion, aunque no son necesarias mientras la declaracion este escrita en su propia linea, es una buena practica, si se desea mas de una declaracion en una linea, entonces debes separarlas con punto y coma.
  • Bloques{} son segmentos de codigo que estan encerrados en llaves, cuando las usamos estamos creando un contexto local.
  • Coma (,) se utiliza para separar elementos en listas, como en arrays, o para separar propiedades en objetos.

Variables

Una variable se refiere a algo que varia o puede variar (una incognita) representa un valor cambiante, en matematicas se representan con algun simbolo, en programacion representa un espacio de memoria donde se guarda algun tipo de dato, se usan palabras clave para declararlas y se identifican con algun nombre claro que describa el dato que almacena.

Palabras clave en variables
  • let se usa para declarar variables, permite asignar su valor durante la ejecución del programa, y tambien volver a reasignarlo en cualquier momento. No permite redeclarar la misma variable. Tiene alcance de bloque (scope block) esto significa que cuando se define en un bloque {} solo existira dentro del bloque donde fue definida, como dentro de un if, for, o una función.
  • const se usa para declarar variables que sean constantes a lo largo del codigo, eso quiere decir que el valor de la variable no puede ser alterado. No permite redeclarar la misma variable, ni reasignarle otro valor, la asignacion se debe definir al momento de declararla en la misma linea, lo que significa que no podemos asig-narle el valor luego.

  • var se usa para declarar variables, tiene alcanze global en el documento, Se pueden redeclarar y reasignar. Normalmente no se suele utilizar para evitar errores de scope, ya que podria alterar el comportamiento de otras variables que esten definidas en otros bloques si llevan el mismo nombre.

Contexto de ejecucion

Un contexto de ejecución es un entorno en el que se ejecuta el código JavaScript. Contiene información sobre el código que se está ejecutando, como las variables declaradas, las funciones y el objeto global. Cada vez que se invoca una función, se crea un nuevo contexto de ejecución.

  • Contexto global se refiere al alcanze que tienen todas las variables y codigo que esta por fuera de funciones o bloques.
  • Contexto local se refiere al alcanze que tienen las variables y codigo que estan dentro de bloques o funciones.

scope chain

La "cadena de alcance" es el mecanismo que utiliza JavaScript para determinar qué variables están disponibles para una instrucción en particular. Se busca en una secuencia de contextos de ejecución hasta encontrar la variable con el nombre correspondiente. Un contexto local puede acceder a variables en contexto global, pero no al revés. La búsqueda de variables funciona de adentro hacia afuera, asegurándose de que no haya acceso entre contextos locales iguales.

let mensajeGlobal = "Hola desde afuera"

function mensajes() {
  let mensajeBloque = "Hola desde el bloque"
  var mensajeVar = "Hola soy var"
  console.log(mensajeGlobal) // obtiene mensajeGlobal globalmente
  console.log(mensajeBloque) // obtiene mensajeBloque localmente
  console.log(mensajeVar) // obtiene mensajeVar localmente
}

  • Alcance: Con let la variable mensajeBloque solo vive dentro del bloque de la funcion. Con var la variable mensajeVar esta disponible fuera del bloque, podriamos necesitar ese nombre de variable en otro bloque pero no podriamos usarlo ya que se sobreescribiria.
  • Contexto local: Desde adentro del bloque estan disponibles las variables locales como: mensajeBloque y mensajevar, pero ademas tiene acceso a variables fuera del bloque, osea a las globales como: mensaGlobal.
  • Contexto global: Desde afuera del bloque estan disponible las variables Globales como: mensajeGlobal pero no se tiene acceso a la variable dentro del bloque como mensajeBloque, y al no encontrarla se detendria el flujo de trabajo, ocacionando que no se ejecute el codigo posterior. En este caso lo mandamos a imprimir de ultimo para que no detenga el resto del codigo.

Variables de entorno

Las variables de entorno son valores dinámicos que existen fuera del código de una aplicación y que afectan cómo se ejecutan los programas, proporcionan información configurable como rutas de archivos, configuraciones de conexión, o modos de aplicación, permitiendo adaptar el comportamiento de una app sin modificar su código fuente.

Es crucial para entornos distintos (desarrollo, producción) y para proteger datos sensibles como contraseñas. Por ejemplo, en nuestro entorno de desarrollo conectamos a una base de datos de prueba, mientras que en producción nos conectamos a una base de datos real, en lugar de utilizar la dirección de la base de datos directamente, utilizaremos una variable que tendrá la dirección de la base de datos.

Estas variables se suelen guardar en un archivo llamado .env y en ella se suelen guardar datos que varían dependiendo del entorno, así como nombres de usuario o contraseñas, que son también sensibles a cambios frecuentes y no queremos que estén acoplados al código fuente.

Creando variables de entorno

Como hemos dicho, el primer paso es crear un fichero .env en nuestro raíz del proyecto. En su interior crearemos las variables en mayúsculas (separadas por guión bajo si se necesita) seguido de un = con el valor correspondiente.

Para leer variables de entorno desde NodeJS, podemos utilizar dos formas diferentes. El método loadEnvFile() de node:process o ejecutar el parámetro --env-file desde la línea de comandos. En ambos casos, le pasamos la ruta del fichero .env que contiene las variables de entorno que queremos leer en nuestra aplicación.

El objeto global no necesitas importarlo; está disponible en cualquier parte de tu aplicación Node.js . Mensajes Emergentes

Javascript ofrece entre sus funciones nativas la posibilidad de crear mensajes emergentes. Más específicamente, alerta (alert), confirmación (confirm) y mensajes personalizados (Prompt)

1. Crear un mensaje de alerta con alert()

La función alert() nos permite desde JavaScript crear un mensaje de alerta sin algo más que un botón para aceptar, de modo que el mensaje desaparecerá cuando el usuario presione aceptar o cierre la ventana. Ejemplo: alert("mensaje de alerta");

2. Crear un mensaje de confirmación con confirm()

A diferencia de la función Alert, la función confirm() además de mostrar un mensaje emergente nos permite aceptar o denegar dicho mensaje. para recojer la inmformacion del usuario debemos guardarla en una variable, igualandola a "confirm()" Ejemplo:
let b = confirm("Estas seguro de borrar?")
alert(b)

3. Crear un mensaje personalizado con Prompt()

La función prompt() nos permite enviar un mensaje emergente con una caja de texto de modo que el usuario puede ingresar un valor dentro de ella. En caso de presionar «aceptar» el valor de establecerá, en caso de presionar «cancelar» en valor quedará vacío. Ejemplo: prompt("introduce tu edad") si queremos recojer la informacion que el usuario escribio en el "prompt" debemos guardarla en una variable, si queremos asignar un valor por defecto lo escribimos despues de una coma.
