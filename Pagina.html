JavaScript

JavaScript es un lenguaje de programación que permite hacer que las páginas web sean interactivas, dinámicas y funcionales. Es uno de los tres pilares del desarrollo web junto con HTML y CSS de cualquier página web, los programas creados en este lenguaje se llaman scripts. Estos scripts son texto y no necesitan ser compilados de ninguna forma para funcionar, como otros lenguajes como puede ser Java.

  • En un inicio se penso para ejecutarse en navegadores, aunque hoy en dia va mucho mas alla.
  • Cualquier dispositivo con un "JavaScript engine" o motor de JavaScript puede ejecutarlo.
  • Los navegadores incluyen un JavaScript engine.

Motor de JavaScript

tambien llamado JavaScript engine es el componente de software que ejecuta el código JavaScript. Es como el “cerebro” que traduce lo que escribes en JS a instrucciones que la computadora realmente puede entender y ejecutar. Existen varios motores de JavaScript: V8 = Chrome y Opera ; SpiderMonkey = Firefox , etc. Además, motores como V8 también se usan fuera del navegador, por ejemplo en Node.js y Deno, para ejecutar JavaScript en servidores o entornos locales.

Capacidades de JavaScript en un navegador

con JavaScript en el navegador no tenemos acceso a la memoria o CPU del sistema. aunque con JavaScript usando NodeJS si que tenemos acceso a estas cosas. En el navegador podemos hacer todo lo relacionado a la manipulacion e interaccion con el usuario de la pagina web. Muchos de estos limites dejan de existir cuando ejecutamos JavaScript fuera del navegador, como por ejemplo en el servidor.

  • Se puede hacer:
  • Añadir nuevo HTML a la web, cambiando el contenido, modificando estilos.
  • Reaccionar a las acciones del usuario, clicks, movimientos, teclas pulsadas, etc.
  • Enviar peticiones a servidores remotos, descargar y subir archivos.
  • Escribir y leer cookies, hacer preguntar al usuario, mostrar mensajes.
  • Guardar datos en el lado del cliente para recordar mas tarde.
  • Crear animaciones, juegos, y efectos visuales. Controlar audio, video y otros elementos multimedia.
  • Manipular el DOM (Document Object Model) para cambiar el contenido de la página en tiempo real.
  • No se puede hacer:
  • No podemos escribir o leer ficheros del disco duro
  • No podemos ejecutar programas instalados en el ordenador
  • Podemos interactuar con la camara o microfono del ordenador, pero siempre con confirmacion por parte del usuraio.
  • Las diferentes pestañas no saben de la existencia de otras pestañas. Excepto cuando desde una pestaña de JavaScript abre otra, pero incluso asi tendria que ser del mismo dominio, protocolo y puerto, es decir si desde tu pagina abres una pestaña de Google por ejemplo, no vas a tener acceso, tiene que ser de tu misma web.

Declaraciones

En JavaScript las intrucciones de denominan "declaraciones" estas le dicen al interprete que realice una accion especifica. Es como una linea de ordenes que el navegador ejecuta paso a paso. Cada declaracion puede: crear una variable, definir una funcion, realizar una condicion, iniciar un bucle.

  • Punto y coma (;) este simbolo se usa separar las declaraciones, indicando que hasta alli llega su ejecucion, aunque no son necesarias mientras la declaracion este escrita en su propia linea, es una buena practica, si se desea mas de una declaracion en una linea, entonces debes separarlas con punto y coma.
  • Bloques{} son segmentos de codigo que estan encerrados en llaves, cuando las usamos estamos creando un contexto local.
  • Coma (,) se utiliza para separar elementos en listas, como en arrays, o para separar propiedades en objetos.

Variables

Una variable se refiere a algo que varia o puede variar (una incognita) representa un valor cambiante, en matematicas se representan con algun simbolo, en programacion representa un espacio de memoria donde se guarda algun tipo de dato, se usan palabras clave para declararlas y se identifican con algun nombre claro que describa el dato que almacena.

Palabras clave en variables
  • let se usa para declarar variables, permite asignar su valor durante la ejecución del programa, y tambien volver a reasignarlo en cualquier momento. No permite redeclarar la misma variable. Tiene alcance de bloque (scope block) esto significa que cuando se define en un bloque {} solo existira dentro del bloque donde fue definida, como dentro de un if, for, o una función.
  • const se usa para declarar variables que sean constantes a lo largo del codigo, eso quiere decir que el valor de la variable no puede ser alterado. No permite redeclarar la misma variable, ni reasignarle otro valor, la asignacion se debe definir al momento de declararla en la misma linea, lo que significa que no podemos asig-narle el valor luego.

  • var se usa para declarar variables, tiene alcanze global en el documento, Se pueden redeclarar y reasignar. Normalmente no se suele utilizar para evitar errores de scope, ya que podria alterar el comportamiento de otras variables que esten definidas en otros bloques si llevan el mismo nombre.

Contexto de ejecucion

Un contexto de ejecución es un entorno en el que se ejecuta el código JavaScript. Contiene información sobre el código que se está ejecutando, como las variables declaradas, las funciones y el objeto global. Cada vez que se invoca una función, se crea un nuevo contexto de ejecución.

  • Contexto global se refiere al alcanze que tienen todas las variables y codigo que esta por fuera de funciones o bloques.
  • Contexto local se refiere al alcanze que tienen las variables y codigo que estan dentro de bloques o funciones.

scope chain

La "cadena de alcance" es el mecanismo que utiliza JavaScript para determinar qué variables están disponibles para una instrucción en particular. Se busca en una secuencia de contextos de ejecución hasta encontrar la variable con el nombre correspondiente. Un contexto local puede acceder a variables en contexto global, pero no al revés. La búsqueda de variables funciona de adentro hacia afuera, asegurándose de que no haya acceso entre contextos locales iguales.

let mensajeGlobal = "Hola desde afuera"

function mensajes() {
  let mensajeBloque = "Hola desde el bloque"
  var mensajeVar = "Hola soy var"
  console.log(mensajeGlobal) // obtiene mensajeGlobal globalmente
  console.log(mensajeBloque) // obtiene mensajeBloque localmente
  console.log(mensajeVar) // obtiene mensajeVar localmente
}

  • Alcance: Con let la variable mensajeBloque solo vive dentro del bloque de la funcion. Con var la variable mensajeVar esta disponible fuera del bloque, podriamos necesitar ese nombre de variable en otro bloque pero no podriamos usarlo ya que se sobreescribiria.
  • Contexto local: Desde adentro del bloque estan disponibles las variables locales como: mensajeBloque y mensajevar, pero ademas tiene acceso a variables fuera del bloque, osea a las globales como: mensaGlobal.
  • Contexto global: Desde afuera del bloque estan disponible las variables Globales como: mensajeGlobal pero no se tiene acceso a la variable dentro del bloque como mensajeBloque, y al no encontrarla se detendria el flujo de trabajo, ocacionando que no se ejecute el codigo posterior. En este caso lo mandamos a imprimir de ultimo para que no detenga el resto del codigo.

Variables de entorno

Las variables de entorno son valores dinámicos que existen fuera del código de una aplicación y que afectan cómo se ejecutan los programas, proporcionan información configurable como rutas de archivos, configuraciones de conexión, o modos de aplicación, permitiendo adaptar el comportamiento de una app sin modificar su código fuente.

Es crucial para entornos distintos (desarrollo, producción) y para proteger datos sensibles como contraseñas. Por ejemplo, en nuestro entorno de desarrollo conectamos a una base de datos de prueba, mientras que en producción nos conectamos a una base de datos real, en lugar de utilizar la dirección de la base de datos directamente, utilizaremos una variable que tendrá la dirección de la base de datos.

Estas variables se suelen guardar en un archivo llamado .env y en ella se suelen guardar datos que varían dependiendo del entorno, así como nombres de usuario o contraseñas, que son también sensibles a cambios frecuentes y no queremos que estén acoplados al código fuente.

Creando variables de entorno

Como hemos dicho, el primer paso es crear un fichero .env en nuestro raíz del proyecto. En su interior crearemos las variables en mayúsculas (separadas por guión bajo si se necesita) seguido de un = con el valor correspondiente.

Para leer variables de entorno desde NodeJS, podemos utilizar dos formas diferentes. El método loadEnvFile() de node:process o ejecutar el parámetro --env-file desde la línea de comandos. En ambos casos, le pasamos la ruta del fichero .env que contiene las variables de entorno que queremos leer en nuestra aplicación.

Manipulacion de numeros

En JavaScript, el tipo de dato primitivo number es esencial para el manejo de operaciones numéricas. Aunque existen diversas formas de escribir números, como enteros y decimales, todos se reconocen bajo el mismo tipo de dato: number. Esta unificación facilita la manipulación y el cálculo numérico. Veamos cómo se representan en este lenguaje:

  • Operadores aritmeticos: suma (+) resta(-) multiplicacion(*) divicion(/) modulo(%) exponente(**)
  • metodo toFixed() se usa para formatear un numero con una cantidad especifica de decimales. devolviendo el resultado como una cadena de texto. El numero se redondea si es necesario.
  • Objeto math da acceso a funciones y constantes matematicas.
    • Math.PI es una propiedad del objeto global Math que representa el valor de π (pi) → 3.14159...
    • Math.round(x) es una función que redondea un número al entero más cercano. Si el decimal es 0.5 o mayor, redondea hacia arriba. Si el decimal es menor que 0.5, redondea hacia abajo. ejemplo: Math.round(4.6) → 5
    • Math.floor(x) sirve para redondear un número hacia abajo al entero más cercano. Ejemplo: Math.floor(4.6) → 4
    • Math.ceil(x) es una función que redondea un número hacia arriba al entero más cercano. Ejemplo: Math.ceil(4.2) → 5
    • Math.abs(x) devuelve el valor absoluto de x, es decir, la distancia entre ese número y el cero, sin importar el signo. - Si x es positivo o cero → lo devuelve tal cual.- Si x es negativo → lo convierte en positivo. Ejemplo: Math.abs(-7) → 7
    • Math.pow(Exponente, Potencia) sirve para elevar un número a una potencia, es decir, calcular la base elevada al exponente Potencia (x elevado a y). Alternativa moderna: (**) Ejemplo: Math.pow(2, 3) → 8
    • Math.sqrt(x) devuelve la raíz cuadrada de x, es decir, el número positivo que, al multiplicarse por sí mismo, da como resultado x. Ejemplo: Math.sqrt(9) → 3
    • Math.max(valor1, valor2...) sirve para encontrar el mayor valor entre dos o más números. Es una función muy útil cuando necesitas comparar valores y quedarte con el más grande. Ejemplo: Math.max(1, 5, 3) → 5
    • Math.min(valor1, valor2...) sirve para encontrar el menor valor entre dos o más números. Ejemplo: Math.min(1, 5, 3) → 1
    • Math.random() genera un número pseudoaleatorio entre 0 (incluido) y 1 (excluido). Es decir, cada vez que lo llamas, te da un número decimal. Ejemplo: Math.random() → 0.73...

  • Items Opcional. Los nuevos elementos que se agregarán a la matriz

    El operador de extencion es una herramienta en javascript que expande elementos de un iterable (como un array, objetos, o incluso strings) en lugares donde se esperan multiples elementos.     • new invoca la funcion constructora, y crea un objeto vacio que sera la nueva instancia y esta recibira los argumentos que pasara a las propiedades del "this". Finalmente devuelve el objeto, que podemos guardar en una variable.

    function Nombre(parametros) {
      this.propiedad = parametros
    }

    new Nombre(argumentos)

    function Persona(nombre, edad) {
      this.nombre = nombre
      this.edad = edad
    }

    const persona1 = new Persona("Luis", 30)
    const persona2 = new Persona("Sharon", 31)
    const persona3 = new Persona("Adriana", 28)

    añadir propiedad individual:
    persona1.ciudad = "Merida"

    añadir propiedad y metodo al "prototype":
    Persona.prototype.pais = "Venezuela"
    Persona.prototype.saludar = function() {
      return `Soy ${this.nombre} tengo ${this.edad} y soy de ${this.pais}
    }

    persona1 + saludar()


    persona2 + saludar()


    persona3 + saludar()
    • Estructura
    • function Persona tiene la estructura parecida a una funcion regular, pero la primera letra del nombre de la funcion debe ser mayuscula, recibe parametros que utilizara para la construccion de la nueva instancia.
    • this.nombre y this.edad crean las propiedades "nombre y edad" que seran utilizados en todas las instancias, estos reciben sus valores desde los parametros.
    • Llamado de funcion
    • new Persona invoca la funcion, y crea un objeto vacio con las propiedades del this. Como devuelve un valor, podemos guardarlo en una variable para luego manipular ese objeto.
    • ("Luis", 30) y manda estos argumentos que corresponden en orden a los parametros que recibe la funcion, y son los valores que tomaran las propiedades.
    • Añadir propiedades y metodos
    • Inidividual: para añadir una propiedad a una instancia particular sin afectar a las demas, simplemente accedemos al objeto y añadimos la nueva propiedad: (objeto.propiedadnueva = valor)
    • Generales: para agregar propiedades a todas las instancias: (funcion-constructora.prototype.propiedad y valor).
    • prototype es una propiedad que tienen todas las funciones, el prototipo es el objeto que sera usado por todas las instancias creadas por "new". Con el podemos agregar propiedades y metodos a todas las instancias, pero no seran parte del objeto, sino del "prototype" de la funcion.
    • Persona.prototype.pais y Persona.prototype.saludar no son visibles en el objeto, pero existen en la propiedad prototype de la funcion, cuando accedemos a propiedades o metodos de una instancia, Javascript busca primero en el objeto directamente, si no esta, busca en el "prototype", y los usa en las instancias.

    Clases

    ofrecen una sintaxis diferente a las funciones constructoras, lo que quiere decir que tambien permiten crear una plantilla que sera el molde para crear multiples instancias de un objeto. Esta nueva forma se le llama "sugar syntax" porque resulta mas intuitiva y familiar debido a su parecido con el lenguaje de programacion Java.

    Sintaxis

    Declaracion
    class Nombre{
      constructor(parametros) {
        this.propiedad = parametros
      }
      metodo() {}
    }

    Instancia
    new Nombre(argumentos)

    • class define un plano o plantilla para crear objetos a partir de ahi. Facilita la estructura ya que permite separar claramente las propiedades y los metodos.
    • Nombre indica el nombre de la clase, que por convencion, inicia con mayuscula.
    • llaves{} definen el cuerpo de la clase, en donde se encuentra el metodo constructor y los metodos que definamos.
    • constructor es un metodo especial que se ejecuta al crear una nueva instancia de la clase, se usa para inicializar propiedades.
    • this se refiere al objeto actual, asi se asignan valores a las propiedades del objeto.
    • Metodos se definen como funciones dentro de la clase, no necesitan la palabra "function", pueden acceder al this, osea al contexto de la clase, para usar sus propiedades y valores.
    • new es la palabra clave para crear las intancias de la clase y con sus valores, esta retorna un objeto con las propiedades del this.
    Creacion del prototipo con constructor
  • En Javascript un prototipo es un objeto del cual otros objetos heredan propiedades y metodos, los generan las funciones y clases constructoras al crearlas, que se usa como un molde para crear instancias de objetos.
    • Creacion de la instacia con new
  • cuando nesecitas crear una instacia de un objeto a partir de un clase constructora, usamos (new + nombreClase(valores)) primero new hereda las propiedades y metodos del prototipo creado y luego con los valores especificados, se llena ese "molde" con datos que requiereas. new devolvera un objeto, que es la nueva instacia, y podemos guardarla en una variable para su manipulacion.
    • Herencias con extends y super
  • es la capacidad que tienen las instancias y otras clases de tomar propiedades o metodos (heredarlas) del prototipo de otra funcion. Esto ocurre naturalmente cuando invocamos new para crear una nueva instacia, pero si necesitamos que otra clase que no sea una instacia herede propiedades y metodos de otra, necesitaremos otra sintaxis y palabras clave:

    • extends se usa en clases para crear una herencia entre una clase hija y una clase padre. Es decir, permite que una clase herede propiedades y métodos de otra, lo que significa que hereda el construtor. Otra capacidad es que puede sobreescribir sus metodos con su propia version.

    super se usa dentro de una clase hija (clase que extiende de otra), para acceder a el constructor de la clase padre, o a los metodos. Funciona escribiendo las propiedades o metodos entre parentesis que se quieren heredar: super(propiedad,propiedad) , super.metodo(). Aunque si una clase extiende otra, hereda todos los metodos publicos de la clase padre automaticamente, podemos usar super para llamar metodos especificos.

  • En caso no heredemos una propiedad (argumento), este asumirá un valor de undefined. Al heredad propiedades con super(), debemos mantener el orden de los argumentos declarados y caso quisiéramos ignorar alguno, le asignamos undefined.
    • Metodos get y set

    tambien conocidos como getters y setters son dos metodos de las clases que permiten obtener (get), y asignar (set), valores de las propiedades de una clase de forma segura. En otras palabras son propiedades de acceso que ejecutan una funcion, y deben tener un nomber propio que sera el usado para su invocacion.

    Formato JSON

    JSON (JavaScript Object Notation) es un formato de texto ligero para representar datos estructurados, inspirado en la sintaxis de objetos de JavaScript pero independiente del lenguaje. Se usa para guardar y transmitir información entre aplicaciones (por ejemplo, cliente y servidor) porque es fácil de leer para personas y máquinas.

    JSON se convirtió en el estándar de facto para intercambio de datos en la web porque es compacto, simple de parsear en la mayoría de los lenguajes y encaja muy bien con las APIs y operaciones asíncronas (AJAX) que evitan recargas completas de página.

    Sintaxis

    {
      "nombre": "luis",
      "edad": 32,
      "compras": ["leche","pan"],
      "activo": true
    }

    • Un documento JSON es texto plano; la raíz puede ser un objeto { ... } o un arreglo [ ... ].
    • las propiedades deben ser cadenas con comillas dobles, seguidas por (:)
    • El valor que almacenan puede ser de estos tipos (cadena, número, booleano, null, objeto o arreglo).
    • Las cadenas de texto deben llevar comillas dobles; no se permiten comillas simples.
    Metodos JSON

    En Javascript tenemos una serie de métodos que nos facilitan la tarea de pasar de de Javascript a JSON y viceversa, pudiendo trabajar con contenido de tipo (que contenga un JSON) y objetos Javascript según interese.

    permite pasar el contenido de texto de un JSON a un objeto, la accion de convertir json a objeto javascript se le suele denominar parsear. Es una accion que analiza un string que contiene un json valido y devuelve un objeto javascript con dicha informacion correctamente estructurada.

    JSON.stringify(Objeto)

    realiza la operacion inversa, convertir un objeto javascript a json (contenido de texto con el json en cuestion), tambien puede recibir dos parametros mas: un array que actuara de filtro a la hora de generar el objeto, en el especificamos los atributos del objeto que queremos traer. Por ultimo el tercer parametro que indica el numero de espacios (identacion) que quieres usar para crear el string.

    Estructuras de datos nativas

    set

    Los Set en Javascript son estructuras de datos nativas que almacenan conjuntos de datos. La característica principal es que los datos insertados no se pueden repetir. por lo que si intentamos insertar un valor que ya existe, no se insertará de nuevo Para iniciar un Set, se debe crear una instancia de su clase a partir de un iterable. Generalmente, un iterable es un array.

    Sintaxis

    const set = new Set(iterable)

    Metodos y propiedades Funcion
    add(element) Añade un elemento al conjunto (si no está repetido) y devuelve el set. Muta
    delete(element) Elimina el element del conjunto. Devuelve true si lo eliminó correctamente.
    has(element) Comprueba si element ya existe en el conjunto. Devuelve true si existe.
    clear() Vacía el conjunto completo.
    size Propiedad que devuelve el número de elementos que tiene el conjunto.

    const set = new Set([1,2,3,4,1])

    // añadir
    set.add(5)

    // encadenar
    set.add(2).add(6)

    // eliminar
    set.delete(3)

    set.clear()

    Mensajes Emergentes

    Javascript ofrece entre sus funciones nativas la posibilidad de crear mensajes emergentes. Más específicamente, alerta (alert), confirmación (confirm) y mensajes personalizados (Prompt)

    1. Crear un mensaje de alerta con alert()

    La función alert() nos permite desde JavaScript crear un mensaje de alerta sin algo más que un botón para aceptar, de modo que el mensaje desaparecerá cuando el usuario presione aceptar o cierre la ventana. Ejemplo: alert("mensaje de alerta");

    2. Crear un mensaje de confirmación con confirm()

    A diferencia de la función Alert, la función confirm() además de mostrar un mensaje emergente nos permite aceptar o denegar dicho mensaje. para recojer la inmformacion del usuario debemos guardarla en una variable, igualandola a "confirm()" Ejemplo:
    let b = confirm("Estas seguro de borrar?")
    alert(b)

    3. Crear un mensaje personalizado con Prompt()

    La función prompt() nos permite enviar un mensaje emergente con una caja de texto de modo que el usuario puede ingresar un valor dentro de ella. En caso de presionar «aceptar» el valor de establecerá, en caso de presionar «cancelar» en valor quedará vacío. Ejemplo: prompt("introduce tu edad") si queremos recojer la informacion que el usuario escribio en el "prompt" debemos guardarla en una variable, si queremos asignar un valor por defecto lo escribimos despues de una coma.
