Clases

Una clase es la forma moderna y clara de definir objetos y manejar herencia, introducida en ES6. Aunque internamente sigue usando el sistema de prototipos, la sintaxis de class ofrece un “azúcar sintáctico” que facilita la programación orientada a objetos.

Esta sintaxis es diferente a las funciones constructoras, se escribe con la palabra clave 'class' seguida del nombre, y el bloque que contiene un contructor de la clase. Class tambien permite crear 'plantillas' un modelo que define como crear propiedades (estado) y metodos (comportamiento).

Sintaxis

Declaracion
class Nombre{
  constructor(parametros) {
    this.propiedad = parametros
  }
  metodo() {}
  static metodo() {}
}

Instancia
new Nombre(argumentos)

  • class define un plano o plantilla para crear objetos a partir de ahi. Facilita la estructura ya que permite separar claramente las propiedades y los metodos.
  • Nombre indica el nombre de la clase, que por convencion, inicia con mayuscula.
  • llaves{} definen el cuerpo de la clase, en donde se encuentra el metodo constructor y los metodos que definamos.
  • constructor es un metodo especial que se ejecuta al crear una nueva instancia de la clase, se usa para inicializar propiedades.
  • this se refiere al objeto actual, asi se asignan valores a las propiedades del objeto.
  • Metodos se definen dentro del prototype de la clase, pueden acceder al this, osea al contexto de la clase, para usar sus propiedades y valores. Se comparten entre todos los objetos creados.
  • static metodo los metodos con la palabra clase 'static' pertenecen a la clase en si, no a las instancias.
  • new es la palabra clave para crear las intancias de la clase y con sus valores, esta retorna un objeto con las propiedades del this.

Creacion del prototipo con constructor
  • En Javascript un prototipo es un objeto del cual otros objetos heredan propiedades y metodos, los generan las funciones y clases constructoras al crearlas, que se usa como un molde para crear instancias de objetos.
    • Creacion de la instacia con new
  • cuando nesecitas crear una instacia de un objeto a partir de un clase constructora, usamos (new + nombreClase(valores)) primero new hereda las propiedades y metodos del prototipo creado y luego con los valores especificados, se llena ese "molde" con datos que requiereas. new devolvera un objeto, que es la nueva instacia, y podemos guardarla en una variable para su manipulacion.

    • Abstracción

    La abstracción consiste en extraer los datos y comportamientos de un objeto para crear un molde o clase que luego permita generar instancias concretas de dicho objeto. En el contexto de JavaScript, esta técnica se traduce en crear prototipos a partir de los cuales se pueden instanciar diferentes objetos con características y comportamientos similares.

    En JavaScript, la abstracción se aplica cuando se crean clases o prototipos para encapsular propiedades y métodos comunes a un grupo de objetos. En lugar de utilizar objetos literales y duplicar código al crear cada instancia de objeto, la abstracción permite definir una estructura base que se puede reutilizar fácilmente. Por ejemplo, si tenemos un grupo de estudiantes que comparten atributos y métodos comunes, como nombre, edad, y un método para estudiar, se puede crear una clase Estudiante.

    Encapsulacion

    En programación orientada a objetos es el principio que consiste en ocultar los detalles internos de un objeto y controlar el acceso a sus datos. Se busca que las propiedades y métodos internos estén protegidos, exponiendo solo lo necesario a través de la interfaz.

    • Ocultar implementación: Los detalles internos de cómo funciona un objeto no son accesibles directamente.
    • Control de acceso: Se usan mecanismos como getters, setters o campos privados (#propiedad) para decidir qué datos pueden leerse o modificarse.
    • Interfaz pública: El objeto expone solo los métodos y propiedades que el programador necesita usar, manteniendo el resto protegido.

    Metodos get y set

    tambien conocidos como getters y setters son dos metodos especiales de las clases que permiten controlar como se accede y se modifica una propiedad dentro de una clase o objeto de forma segura: obtener (get), y asignar (set). En otras palabras son propiedades de acceso que ejecutan una funcion, y deben tener un nomber propio que sera el usado para su invocacion.

    Getter get

    Método que se ejecuta cuando se accede a una propiedad. Sirve para leer valores, pero puede incluir lógica adicional (por ejemplo, calcular un valor dinámico). Permite retornar un valor transformado o calculado en lugar del valor almacenado, lo que es útil para mantener la encapsulación y controlar el acceso a los datos internos de una clase.

    Setter set

    Método que se ejecuta cuando se asigna un valor a una propiedad, no retornan un valor, solo asignan el valor a la propiedad, pero con la posibilidad de incluir lógica adicional, como validaciones o transformaciones, antes de asignarlo a la propiedad interna.

    Campos privados

    Los campos privados son propiedades que solo pueden ser accedidas desde dentro de la clase, lo que permite ocultar detalles internos y controlar el acceso a los datos. En JavaScript, se definen usando el prefijo # antes del nombre de la propiedad. Esto garantiza que la propiedad no pueda ser accedida ni modificada directamente desde fuera de la clase.

Esto es un ejemplo de cómo la encapsulación protege los datos internos de una clase y controla su acceso mediante métodos específicos.

    • Acceso a propiedades
    • David.age → acceso crudo, sin lógica.
    • David.userAge → acceso controlado, pasa por el getter, lo que te da la oportunidad de modificar el comportamiento (ejemplo: devolver "Mayor de edad" si age >= 18).
    • Modificación de propiedades
    • David.age = 25 → asignación cruda, sin lógica.
    • David.userAge = 25 → asignación controlada, pasa por el setter, lo que te da la oportunidad de modificar el comportamiento (ejemplo: validar que la edad sea mayor a 18).

    Herencia

    Es la capacidad que tienen las instancias y otras clases de tomar propiedades o metodos (heredarlas) del prototipo de otra funcion. Esto ocurre naturalmente cuando invocamos new para crear una nueva instacia, pero si necesitamos que otra clase que no sea una instacia herede propiedades y metodos de otra, necesitaremos otra sintaxis y palabras clave:

    extends

    Se usa en clases para crear una herencia entre una clase hija y una clase padre. Es decir, permite que una clase herede propiedades y métodos de otra, lo que significa que hereda el construtor. Otra capacidad es que puede sobreescribir sus metodos con su propia version.

    super

    Se usa dentro de una clase hija (clase que extiende de otra), para acceder a el constructor de la clase padre, o a los metodos. Funciona escribiendo las propiedades o metodos entre parentesis que se quieren heredar: super(propiedad,propiedad) , super.metodo(). Aunque si una clase extiende otra, hereda todos los metodos publicos de la clase padre automaticamente, podemos usar super para llamar metodos especificos.

    En caso no heredemos una propiedad (argumento), este asumirá un valor de undefined. Al heredad propiedades con super(), debemos mantener el orden de los argumentos declarados y caso quisiéramos ignorar alguno, le asignamos undefined.

    Polimorfismo

    Es la capacidad de un objeto de tomar muchas formas, es decir, que una misma función o método pueda comportarse de diferentes maneras según el contexto. En JavaScript, esto se logra a través de la herencia y la sobreescritura de métodos en las clases.

    Por ejemplo, si tenemos una clase Animal con un método hacerSonido(), y luego creamos clases específicas como Perro y Gato que extienden de Animal, cada una puede implementar su propia versión de hacerSonido() para producir un sonido diferente (ladrido para el perro, maullido para el gato). Esto permite que el mismo método haga cosas diferentes dependiendo del tipo de objeto que lo esté utilizando, lo que es una manifestación clara del polimorfismo.

    Metodos y Atributos Estaticos

    Los métodos estáticos en JavaScript son herramientas de clase, no de instancia. Se usan para lógica compartida, utilidades y patrones de diseño como fábricas o contadores. No requieren instanciación y se acceden directamente a través de la clase, lo que los hace ideales para funciones que no dependen del estado de un objeto específico. Son fundamentales para organizar el código y promover la reutilización.

    static

    La palabra clave static se utiliza para definir métodos estáticos dentro de una clase en JavaScript. Estos métodos no requieren una instancia de la clase para ser llamados, ya que pertenecen a la clase en sí misma. Se acceden directamente a través del nombre de la clase, lo que los hace ideales para funciones que no dependen del estado de un objeto específico. Los métodos estáticos son útiles para organizar el código y promover la reutilización, especialmente para funciones de utilidad o lógica compartida entre todas las instancias de la clase.
