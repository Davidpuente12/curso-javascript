Objetos

Los objetos en JavaScript son estructuras de datos que permiten almacenar información de manera más compleja que los arreglos o cadenas. A diferencia de estos últimos, que son sencillamente listas de elementos, los objetos están compuestos por "propiedades" y "valores" asociados.

Funciona como un almacen en donde podemos guardar "variables" y sus valores, o incluso funciones. Este concepto es crucial para representar entidades del mundo real en el ambito digital, permitiendo abstraer y estructurar informacion de forma organizada.

Sintaxis

let carro = {
  propiedad : "valor" ,
  metodo : function() {}
}

  • llaves{} abren un segmento o bloque en donde se almacenan las propiedades, valores y metodos, que representan al objeto en si.
  • Propiedad las propiedades son como una variable dentro del objeto, que pueden contener cualquier tipo de dato, estos son los valores. Cada propiedad representa un atributo del objeto.
  • Valor cualquier tipo de dato asociado a la propiedad, que puede ser un número, string, booleano, función, u otro objeto.
  • Metodos son las acciones o comportamientos que el objeto puede realizar. Estos se definen por medio de funciones.

Acceder a los valores

para acceder a los valores dentro de las propiedades tenemos dos fomas, una es a traves de la notacion de punto: (objeto.propiedad) , (objeto.metodo). Y la otra forma es a travez de la notacion con corchetes: (objeto["propiedad"]).

  • objeto.propiedad muestra el valor de la propiedad directamente. (util para obtener nombres simples).
  • objeto["propiedad"] muestra el valor de la propiedad, permite encontrar keys con nombres no validos para la notacion con punto, como: (nombres-con-guiones). Es mejor practica usarla cuando la key que se quiere obtener viene desde una variable externa.
  • objeto.metodo muestra el metodo, osea la funcion completa. para ejecutar funciones que estan dentro de objetos, la llamamos mediante parentesis ().
Metodo Object

La clase Object tiene métodos que devuelven un array de las claves, valores o pares clave-valor de las propiedades propias y enumerables del objeto.

  • Object.keys(objeto) Devuelve un array con los nombres (claves) de las propiedades propias y enumerables.
  • Object.values(objeto) Devuelve un array con los valores de las propiedades propias y enumerables.
  • Object.entries(objeto) Devuelve un array de arrays [clave, valor] (pares clave-valor).
  • Object.fromEntries(array) Este metodo devuelve un objeto a partir de un array donde sus elementos son las entries en forma ["propiedad", "valor"]. Se considera la operación inversa de Object.entries.
  • Object.assign({})

    Los objetos literales en JavaScript son mutables: puedes añadir o actualizar en cualquier momento.

    Optional Chaining

    Cuando intentas acceder a propiedades de un objeto que no existen, JavaScript te retornará undefined. Al acceder a una propiedad más profunda de un objeto, que previamente fue evaluada como undefined, el programa se detendrá y mostrará un error de tipo.

    El encadenamiento opcional (?) es una forma segura de realizar comprobaciones de acceso a propiedades de objetos anidados, este operador toma la referencia a su izquierda y verifica si no está definida o es nula. Si la referencia es cualquiera de estos valores nulos, las comprobaciones se detendrán y devolverán un valor indefinido. Se debe utilizar únicamente cuando probablemente un valor no exista.

    Desestructuracion de Objetos

    Consiste en extraer las propiedades de un objeto en variables individuales que deben tener el mismo nombre del objeto, estas variables hacen referencia a las propiedades del objeto. Tambien podemos enviar un objeto como argumento de una funcion gracias a la desesctrucruracion, En los parametros la funcion debe esperar el objeto y sus propiedades, de esta manera se obtienen directamente.

    Funcion constructora

    una función constructora es una forma de crear múltiples objetos con estructura similar sin necesidad de duplicar código. Funciona como una plantilla, que tiene una estructura y comportamiento. La función constructora tiene ciertas caracteristicas:

    • Estructura como una función regular: Aunque tiene nombre específico, la estructura es similar a una función regular, pasando los datos necesarios como parámetros.
    • Nombre Iniciamos el nombre de la función constructora con la primera letra en mayúscula para indicar que esta función se utilizará para crear objetos, y que se debe invocar con "new".
    • parametros una o mas variables que definen los datos (valores) necesarios para construir la instancia.
    • this se usa para asignar propiedades especificas del nuevo objeto.
    • new invoca la funcion constructora, y crea un objeto vacio que sera la nueva instancia y esta recibira los argumentos que pasara a las propiedades del "this". Finalmente devuelve el objeto, que podemos guardar en una variable.

    Sintaxis

    function Nombre(parametros) {
      this.propiedad = parametros
    }

    new Nombre(argumentos)

    function Persona(nombre, edad) {
      this.nombre = nombre
      this.edad = edad
    }

    const persona1 = new Persona("Luis", 30)
    const persona2 = new Persona("Sharon", 31)
    const persona3 = new Persona("Adriana", 28)

    añadir propiedad individual:
    persona1.ciudad = "Merida"

    añadir propiedad y metodo al "prototype":
    Persona.prototype.pais = "Venezuela"
    Persona.prototype.saludar = function() {
      return `Soy ${this.nombre} tengo ${this.edad} y soy de ${this.pais}
    }

    persona1 + saludar()


    persona2 + saludar()


    persona3 + saludar()
    • Estructura
    • function Persona tiene la estructura parecida a una funcion regular, pero la primera letra del nombre de la funcion debe ser mayuscula, recibe parametros que utilizara para la construccion de la nueva instancia.
    • this.nombre y this.edad crean las propiedades "nombre y edad" que seran utilizados en todas las instancias, estos reciben sus valores desde los parametros.
    • Llamado de funcion
    • new Persona invoca la funcion, y crea un objeto vacio con las propiedades del this. Como devuelve un valor, podemos guardarlo en una variable para luego manipular ese objeto.
    • ("Luis", 30) y manda estos argumentos que corresponden en orden a los parametros que recibe la funcion, y son los valores que tomaran las propiedades.
    • Añadir propiedades y metodos
    • Inidividual: para añadir una propiedad a una instancia particular sin afectar a las demas, simplemente accedemos al objeto y añadimos la nueva propiedad: (objeto.propiedadnueva = valor)
    • Generales: para agregar propiedades a todas las instancias: (funcion-constructora.prototype.propiedad y valor).
    • prototype es una propiedad que tienen todas las funciones, el prototipo es el objeto que sera usado por todas las instancias creadas por "new". Con el podemos agregar propiedades y metodos a todas las instancias, pero no seran parte del objeto, sino del "prototype" de la funcion.
    • Persona.prototype.pais y Persona.prototype.saludar no son visibles en el objeto, pero existen en la propiedad prototype de la funcion, cuando accedemos a propiedades o metodos de una instancia, Javascript busca primero en el objeto directamente, si no esta, busca en el "prototype", y los usa en las instancias.

    Clases

    ofrecen una sintaxis diferente a las funciones constructoras, lo que quiere decir que tambien permiten crear una plantilla que sera el molde para crear multiples instancias de un objeto. Esta nueva forma se le llama "sugar syntax" porque resulta mas intuitiva y familiar debido a su parecido con el lenguaje de programacion Java.

    Sintaxis

    Declaracion
    class Nombre{
      constructor(parametros) {
        this.propiedad = parametros
      }
      metodo() {}
    }

    Instancia
    new Nombre(argumentos)

    • class define un plano o plantilla para crear objetos a partir de ahi. Facilita la estructura ya que permite separar claramente las propiedades y los metodos.
    • Nombre indica el nombre de la clase, que por convencion, inicia con mayuscula.
    • llaves{} definen el cuerpo de la clase, en donde se encuentra el metodo constructor y los metodos que definamos.
    • constructor es un metodo especial que se ejecuta al crear una nueva instancia de la clase, se usa para inicializar propiedades.
    • this se refiere al objeto actual, asi se asignan valores a las propiedades del objeto.
    • Metodos se definen como funciones dentro de la clase, no necesitan la palabra "function", pueden acceder al this, osea al contexto de la clase, para usar sus propiedades y valores.
    • new es la palabra clave para crear las intancias de la clase y con sus valores, esta retorna un objeto con las propiedades del this.
    Creacion del prototipo con constructor
  • En Javascript un prototipo es un objeto del cual otros objetos heredan propiedades y metodos, los generan las funciones y clases constructoras al crearlas, que se usa como un molde para crear instancias de objetos.
    • Creacion de la instacia con new
  • cuando nesecitas crear una instacia de un objeto a partir de un clase constructora, usamos (new + nombreClase(valores)) primero new hereda las propiedades y metodos del prototipo creado y luego con los valores especificados, se llena ese "molde" con datos que requiereas. new devolvera un objeto, que es la nueva instacia, y podemos guardarla en una variable para su manipulacion.
    • Herencias con extends y super
  • es la capacidad que tienen las instancias y otras clases de tomar propiedades o metodos (heredarlas) del prototipo de otra funcion. Esto ocurre naturalmente cuando invocamos new para crear una nueva instacia, pero si necesitamos que otra clase que no sea una instacia herede propiedades y metodos de otra, necesitaremos otra sintaxis y palabras clave:

    • extends se usa en clases para crear una herencia entre una clase hija y una clase padre. Es decir, permite que una clase herede propiedades y métodos de otra, lo que significa que hereda el construtor. Otra capacidad es que puede sobreescribir sus metodos con su propia version.

    super se usa dentro de una clase hija (clase que extiende de otra), para acceder a el constructor de la clase padre, o a los metodos. Funciona escribiendo las propiedades o metodos entre parentesis que se quieren heredar: super(propiedad,propiedad) , super.metodo(). Aunque si una clase extiende otra, hereda todos los metodos publicos de la clase padre automaticamente, podemos usar super para llamar metodos especificos.

  • En caso no heredemos una propiedad (argumento), este asumirá un valor de undefined. Al heredad propiedades con super(), debemos mantener el orden de los argumentos declarados y caso quisiéramos ignorar alguno, le asignamos undefined.
    • Metodos get y set

    tambien conocidos como getters y setters son dos metodos de las clases que permiten obtener (get), y asignar (set), valores de las propiedades de una clase de forma segura. En otras palabras son propiedades de acceso que ejecutan una funcion, y deben tener un nomber propio que sera el usado para su invocacion.

    Formato JSON

    JSON (JavaScript Object Notation) es un formato de texto ligero para representar datos estructurados, inspirado en la sintaxis de objetos de JavaScript pero independiente del lenguaje. Se usa para guardar y transmitir información entre aplicaciones (por ejemplo, cliente y servidor) porque es fácil de leer para personas y máquinas.

    JSON se convirtió en el estándar de facto para intercambio de datos en la web porque es compacto, simple de parsear en la mayoría de los lenguajes y encaja muy bien con las APIs y operaciones asíncronas (AJAX) que evitan recargas completas de página.

    Sintaxis

    {
      "nombre": "luis",
      "edad": 32,
      "compras": ["leche","pan"],
      "activo": true
    }

    • Un documento JSON es texto plano; la raíz puede ser un objeto { ... } o un arreglo [ ... ].
    • las propiedades deben ser cadenas con comillas dobles, seguidas por (:)
    • El valor que almacenan puede ser de estos tipos (cadena, número, booleano, null, objeto o arreglo).
    • Las cadenas de texto deben llevar comillas dobles; no se permiten comillas simples.
    Metodos JSON

    En Javascript tenemos una serie de métodos que nos facilitan la tarea de pasar de de Javascript a JSON y viceversa, pudiendo trabajar con contenido de tipo (que contenga un JSON) y objetos Javascript según interese.

    JSON.parse(JSONstring)

    permite pasar el contenido de texto de un JSON a un objeto, la accion de convertir json a objeto javascript se le suele denominar parsear. Es una accion que analiza un string que contiene un json valido y devuelve un objeto javascript con dicha informacion correctamente estructurada.

    JSON.stringify(Objeto)

    realiza la operacion inversa, convertir un objeto javascript a json (contenido de texto con el json en cuestion), tambien puede recibir dos parametros mas: un array que actuara de filtro a la hora de generar el objeto, en el especificamos los atributos del objeto que queremos traer. Por ultimo el tercer parametro que indica el numero de espacios (identacion) que quieres usar para crear el string.

    const json = '{"nombre": "luis", "apellido": "Puente" , "edad": 32}'
