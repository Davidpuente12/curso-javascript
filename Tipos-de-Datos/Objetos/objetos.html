Objetos

Los objetos en JavaScript son estructuras de datos que permiten almacenar información de manera más compleja que los arreglos o cadenas. A diferencia de estos últimos, que son sencillamente listas de elementos, los objetos están compuestos por "propiedades" y "valores" asociados.

Funciona como un almacen en donde podemos guardar "variables" y sus valores, o incluso funciones. Este concepto es crucial para representar entidades del mundo real en el ambito digital, permitiendo abstraer y estructurar informacion de forma organizada.

Sintaxis

let carro = {
  propiedad : "valor" ,
  metodo : function() {}
}

  • llaves{} abren un segmento o bloque en donde se almacenan las propiedades, valores y metodos, que representan al objeto en si.
  • Propiedad las propiedades son como una variable dentro del objeto, que pueden contener cualquier tipo de dato. Cada propiedad representa un atributo del objeto.
  • Valor cualquier tipo de dato asociado a la propiedad, que puede ser un número, string, booleano, función, u otro objeto.
  • Metodos son las acciones o comportamientos que el objeto puede realizar. Estos se definen por medio de funciones.
Obtener elementos

para acceder a los valores dentro de las propiedades tenemos dos fomas, una es a traves de la notacion de punto: (objeto.propiedad) , (objeto.metodo). Y la otra forma es a travez de la notacion con corchetes: (objeto["propiedad"]).

  • objeto.propiedad muestra el valor de la propiedad directamente. (util para obtener nombres simples).
  • objeto["propiedad"] muestra el valor de la propiedad, permite encontrar keys con nombres no validos para la notacion con punto, como: (nombres-con-guiones). Es mejor practica usarla cuando la key que se quiere obtener viene desde una variable externa.
  • objeto.metodo muestra el metodo, osea la funcion completa. para ejecutar funciones que estan dentro de objetos, la llamamos mediante parentesis ().
Añadir propiedades o metodos

Al igual que asignamos valores a las variables, en los objetos se hace mediante el operador de asignacion: (objeto.propiedadnueva = valor) . primero se escribe el objeto y seguido del punto el nuevo atributo o metodo del mismo. Si la propiedad no existe, se crea con el valor especificado, si ya existe, se actualiza con el nuevo valor.

Para añadir un nuevo método, se asigna una función a una nueva propiedad del objeto. Esto permite que el objeto tenga nuevas capacidades o comportamientos definidos por esa función.

objeto.propiedad = valor || objeto.metodo = function
Eliminar propiedades o metodos

delete es un operador que sirve para eliminar propiedades o métodos de un objeto. Devuelve true si la operacion fue exitosa, aunque no siempre significa que la propiedad desaparecio (hay casos especiales).

delete objeto.propiedad/metodo

Metodos de los Objetos

Los métodos de los objetos son funciones internas que están asociadas directamente a un objeto y que permiten interactuar, manipular o consultar sus propiedades y comportamientos.

En otras palabras, son operaciones que un objeto puede ejecutar sobre sí mismo o sobre sus datos, como verificar si tiene una propiedad, obtener sus claves, copiarlo, congelarlo, etc.

hasOwnProperty es una función propia de los objetos en JavaScript que sirve para verificar si un objeto tiene una propiedad específica como propiedad directa, es decir, que no fue heredada de su prototipo.

objeto.hasOwnProperty(propiedad)
Objetos anidados
  • entre los tipos de datos que puede contener un objeto, incluye la anidacion de otros objetos, que se convierten en una propiedad mas del objeto, para acceder a sus atributos, usamos la notacion de punto: (objeto.propiedad.propiedad). automovil.direccion.calle muestra el valor de la propiedad, del objeto anidado.
    • Objetos Literales

    Es la forma directa de crear un objeto usando llaves {} para agrupar pares clave-valor. Es la manera más común de definir objetos sin necesidad de usar clases o funciones constructoras. Los objetos literales en JavaScript son mutables: puedes añadir o actualizar en cualquier momento. Son ideales para representar datos estructurados de forma rápida y sencilla, como configuraciones, respuestas de APIs o cualquier entidad que no requiera métodos complejos.

    Property Shorthand (abreviatura)

    consiste en crear objetos a partir de variables que ya estan definidas por fuera, esto evita la necesidad de repetir variables internas que guarden valores externos, sino que javascript creara la propiedad a partir del nombre de la variable con su respectivo valor.

    Tambien se puede usar esta abreviatura para enviar objetos como argumentos a funciones, en este caso se debe esperar un objeto con las propiedades que se quieren obtener, y al enviar el objeto como argumento, se puede usar la abreviatura para no repetir cada propiedad del objeto.

    Optional Chaining

    Cuando intentas acceder a propiedades de un objeto que no existen, JavaScript te retornará undefined. Al acceder a una propiedad más profunda de un objeto, que previamente fue evaluada como undefined, el programa se detendrá y mostrará un error de tipo.

    El encadenamiento opcional (?) es una forma segura de realizar comprobaciones de acceso a propiedades de objetos anidados, este operador toma la referencia a su izquierda y verifica si no está definida o es nula. Si la referencia es cualquiera de estos valores nulos, las comprobaciones se detendrán y devolverán un valor indefinido. Se debe utilizar únicamente cuando probablemente un valor no exista.

    Desestructuracion de Objetos

    Consiste en extraer las propiedades de un objeto en variables individuales que deben tener el mismo nombre del objeto, estas variables hacen referencia a las propiedades del objeto. Tambien podemos enviar un objeto como argumento de una funcion gracias a la desesctrucruracion, En los parametros la funcion debe esperar el objeto y sus propiedades, de esta manera se obtienen directamente.

    const automovil = {
      marca: "Toyota",
      modelo: "Corolla",
      direccion: {calle: 200, carrera: 22},
    }

    Desestrucrurar
    const {marca, modelo, direccion:{calle, carrera}} = automovil

    cambiar nombres
    const {marca: brand} = automovil

    enviar a funcion
    function auto({marca, modelo, direccion}) {
      return ' $(marca) $(modelo) $(direccion.calle) '
    }

    Función constructora Object

    Es una función constructora global en JavaScript. Es la base de la mayoría de los objetos en JavaScript, ya que todos los objetos creados (literales, instancias de clases, arrays, funciones, etc.) heredan directamente o indirectamente de Object.prototype. Proporciona métodos estáticos y propiedades que permiten manipular y trabajar con objetos de manera eficiente.

    • Como función constructora, permite crear nuevos objetos.
    • Como objeto global, ofrece métodos estáticos para manipular y controlar objetos.

    Metodos Estaticos

    Los métodos estáticos en JavaScript son herramientas de clase, no de instancia. Se usan para lógica compartida, utilidades y patrones de diseño como fábricas o contadores. No requieren instanciación y se acceden directamente a través de la clase, lo que los hace ideales para funciones que no dependen del estado de un objeto específico. Son fundamentales para organizar el código y promover la reutilización.

    Métodos estáticos Descripción
    Object.keys(objeto) Devuelve un array con los nombres (claves) de las propiedades propias y enumerables.
    Object.values(objeto) Devuelve un array con los valores de las propiedades propias y enumerables.
    Object.entries(objeto) Devuelve un array de arrays [clave, valor] (pares clave-valor).
    Object.fromEntries(array) Devuelve un objeto a partir de un array donde sus elementos son las entries en forma ["propiedad", "valor"]. Se considera la operación inversa de Object.entries.
    Object.assign({}) Asigna propiedades de uno o más objetos a otro objeto.

    Metodos de prototype

    Los métodos de prototype son funciones que pueden ser utilizadas por todas las instancias de un objeto creado a partir de una función constructora. Estos métodos se definen en el objeto prototype de la función constructora, lo que permite que todas las instancias compartan el mismo código para esos métodos, en lugar de tener una copia de la función en cada instancia.

    Esto es especialmente útil para ahorrar memoria y mejorar el rendimiento, ya que las instancias pueden acceder a los métodos a través de la cadena de prototipos. Además, los métodos de prototype permiten la herencia, lo que significa que las instancias pueden heredar métodos de su función constructora y de cualquier función constructora en la cadena de prototipos.

    Métodos de prototype Descripción
    hasOwnProperty(prop) Verifica si el objeto tiene una propiedad específica como propiedad directa, es decir, que no fue heredada de su prototipo.
    isPrototypeOf(prop) Verifica si un objeto es parte de la cadena de prototipos de otro objeto.
    propertyIsEnumerable(prop) Verifica si una propiedad es enumerable, es decir, si se puede iterar sobre ella con un bucle for...in.
    toString() Devuelve una cadena que representa al objeto. Es comúnmente sobrescrito para proporcionar una representación personalizada del objeto.
    valueOf() Devuelve el valor primitivo del objeto. Al igual que toString, a menudo se sobrescribe para proporcionar un valor personalizado.

    const usuario = new Object();

    usuario.nombre = "David";
    usuario.apellido = "Puente";
    usuario.edad = 30;
    usuario.hobbies = ["programar", "jugar", "correr"];
    usuario.saludar = function () {
      return `Hola, mi nombre es ${this.nombre} ${this.apellido}.`;
    };

    Propiedades Descripción
    Object.prototype Es un objeto que se utiliza como plantilla para crear nuevos objetos. Contiene métodos y propiedades que son heredados por los objetos creados a partir de él.
    Object.constructor Es una referencia a la función constructora que creó el objeto. Por defecto, apunta a la función Object, pero puede ser modificado.
    Object.length Es una propiedad que indica el número de argumentos esperados por la función constructora Object. En este caso, su valor es 1, ya que espera un solo argumento (el objeto a crear).
    Object.name Es una propiedad que contiene el nombre de la función constructora. En este caso, su valor es "Object".
